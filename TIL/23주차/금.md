# AI

## 내비게이션 인보커

Navigation System은 에이전트가 Navigation Mesh를 사용하여 레벨 내에서 경로를 찾고 이동하게 한다.
Navigation System에는 Navigation Mesh의 다양한 Runtime Generation 메서드뿐 아니라 특정 타깃을 중심으로 부분적인 내비게이션을 생성하는 방법도 있다.

Navigation Invoker는 런타임에서 에이전트 주변에 내비게이션 메시를 생성하는 블루프린트 액터 컴포넌트, 내비게이션 인보커를 사용하면 내비게이션 메시를 에디터에서 빌드할 필요가 없어지며 런타임에서 생성되는 타일 수를 제한할 수 있다.

## MoveToActor와 MoveToLocation

AIController -> MoveToLocation은 AI시스템에서 특정 위치로 이동시키는 핵심 함수
MoveToActor는 Location기능에 몇가지 더 얹은 것

공통점 : AI가 특정 오브젝트로 이동하게 만드는것
차이점 : 타겟 오브젝트가 Actor냐 Location이냐의 차이, Actor를 타겟 오브젝트로 삼느냐, Location(특정위치)를 타겟 오브젝트로 삼느냐가 핵심 차이점이며 Actor의 경우 Actor가 실시간으로 이동하면 추적이 가능하고, Location의 경우 특정 좌표로만 이동

MoveToActor는 Location이 실시간으로 변함, 결국 Actor의 위치도 Location이기 때문에 Location을 실시간 업데이트하는것과 같다. 그래서 MoveToActor가 더 비용이 비싸다.

## RVO

RVO(Reciprocal Velocity Obstacles)는 움직이는 객체들이 서로의 속도와 방향을 고려하여 충돌을 피하는 방법
사람붐비는곳에서 서로 피해서 걷는것처럼

경로찾기 기능은 움직이지 않는 오브젝트를 우회하도록 디자인하는거였는데 이와달리 RVO는 움직이는 액터를 회피할 수 있도록 하는 기능, 경로 재계산없이 속도와 방향만 조정하여 실시간 회피가 가능하므로 다수의 액터가 있을 때 효율적, 자연스러운 움직임 제공하는게 장점
