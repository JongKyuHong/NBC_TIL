# 알고리즘

## 개념

- A스타 알고리즘 : 목적지를 향한 다익스트라
  - 다익스트라 + 휴리스틱 함수로 목표 방향성 부여
  - 휴리스틱 : 현재->목표까지 예상 비용(맨해튼/유클리드 거리 등)
  - 게임, 내비게이션에서 널리 사용

- 플로이드워셜 : 모든 노드 쌍 간의 최단 경로
  - 3중 반복문으로 구현되는 DP접근법
  - 시간 복잡도 O(V^3), 모든 노드 사이의 거리를 한 번에 계산
  - 음수 가중치 허용 -> 음수 사이클 없어야 함 -> 계속 최단거리가 갱신될테니까

- 벨만 포드 : 음수 가중치도 처리 가능
  - 모든 간선을 V-1번 반복 확인
  - 다익스트라보다 느리지만 O(VE) 음수 가중치 처리 가능
  - 음수 사이클 감지에도 활용

- 최소 신장 트리(MST) : 최소 비용으로 모든 노드 연결
  - 크루스칼 : 가장 작은 가중치 간선부터 사이클 없이 선택 (Union-Find 활용)
  - 프림 : 특정 정점에서 시작해 트리를 키워나감(다익스트라와 유사)

- 세그먼트 트리 : 구간 쿼리 처리의 강자
  - 구간 합, 최댓값, 최솟값 등을 O(log n)에 처리
  - 구간 업데이트도 빠르게 지원

- 트라이 : 문자열 검색 전문가
  - 문자열 집합 내 검색을 O(m)시간에 수행(m: 문자열 길이)
  - 자동완성, 사전 검색에 활용

- Union-Find(Disjoint Set) : 집합의 합치기와 찾기
  - 원소가 어떤 집합에 속하는지 빠르게 판단
  - 집합 병합을 거의 상수 시간에 처리(경로 압축 + 랭크 최적화 적용 시)
  - 크루스칼 알고리즘, 사이클 판별에 활용

### 배운점

- 열이 4까지 밖에없어서 그냥 하드코딩으로 풀었는데 실제로는 for문 여러개 쓰면 가능하다.
- 마지막에 answer = max({dp[N-1][0],dp[N-1][1],dp[N-1][2],dp[N-1][3]}); 부분으로 했는데 이쪽도 algorithm헤더에 있는 max_element를 쓰면 쉽게 할 수 있다고 한다.

```c++
#include <algorithm>
answer = *max_element(dp[N-1].begin(), dp[N-1].end());
```

- max_element는 가장 큰값이 들어있는 iterator를 반환한다고 한다 그래서 \*를붙여서 값을 꺼내야 함
