# 증강 시스템 UI 연동 정리

## 1. 아키텍처 결정사항

### UI ↔ 게임플레이 통신 방식

**결론: 컴포넌트 델리게이트 방식 채택**

처음에 `GameplayMessageRouter`(태그 기반 pub/sub)를 검토했으나, UE 5.7에서 해당 플러그인이 제거되어 있음. 대신 `DECLARE_DYNAMIC_MULTICAST_DELEGATE`를 사용하는 패턴으로 확정.

```
게임플레이 → UI: SelectionComponent의 델리게이트 → 위젯 구독
UI → 서버:      Server RPC (Server_Choose, Server_RerollCard)
```

위젯이 `NativeConstruct`에서 `PC->FindComponentByClass<UNSAugmentSelectionComponent>()`로 컴포넌트를 찾아 `OnOfferPresented` / `OnOfferClosed`를 `AddDynamic`으로 구독.

## 2. 입력 시스템 연동 (`NSInputBinderComponent`)

### 카드 선택 1/2/3 — Enhanced Input 일원화

이전에 `NSPlayerController`에 `SelectAugmentCard1/2/3` 함수 3개 + raw `BindKey` 3줄이 있던 구조를 전면 교체.

- 태그 추가: `Input.Augment.Card1/2/3`, `Input.Augment.Reroll` (`NSGameplayTags_Input.h`)
- `NSInputBinderComponent`에 `Input_AugmentAction(FGameplayTag InputTag)` 단일 핸들러로 통합 — 태그로 카드 선택/리롤 분기
- `NSAugmentationWidget::NativeOnKeyDown` 오버라이드 **완전 제거** (EI로 전부 이전)

### 증강 창 중 캐릭터 입력 차단 — IMC 스위칭 방식

`FInputModeUIOnly`를 쓰면 폰에 바인딩된 Enhanced Input(카드 선택 IA)까지 죽음. 해결책은 **IMC 레이어 교체**.

```
증강 창 열림 → EnterAugmentInputMode()
  - 현재 active IMC 태그셋 CachedInputModeTagsBeforeAugment에 저장
  - Play IMC 제거, IMC_Augment 활성화
  → 이동/시점/공격 입력 차단, 1/2/3/R만 살아남

증강 창 닫힘 → ExitAugmentInputMode()
  - CachedInputModeTagsBeforeAugment로 복원
```

사망 관전 모드(`NSDeathSpectatorPawn::SetActiveInputModeTags`)와 동일한 패턴.

## 3. UI 흐름 (데이터 → 화면)

```
서버: Server_Choose / Server_RerollCard
  → RollAndPresent()
  → Client_PresentOffer(OfferIds, RerollCost)     ← 클라이언트 RPC
    → OnOfferPresented 델리게이트 브로드캐스트
      → HandleOfferPresented(OfferIds, RerollCost)
        → CurrentOfferIds 저장
        → CreateChoiceCard(N) — 카드 위젯 재생성
        → 아이콘 소프트포인터 수집 → RequestAsyncLoad()
          → OnIconsLoaded()
            → PopulateOfferCards()   — Definition에서 이름/설명/아이콘 채우기
            → ShowAugmentation()     — IMC 스위칭 + Visible

서버: 증강 선택 완료 → Client_CloseOffer()
  → OnOfferClosed 델리게이트
    → HandleOfferClosed()
      → IconLoadHandle 취소
      → HideAugmentation()   — IMC 복원 + Collapsed
```

## 4. 자산 로딩 전략

**아이콘만 클라이언트에서 비동기 로드**, GE/GA는 서버 `ApplyAugment`에서 로드.

- `HandleOfferPresented`에서 `Def->Icon`(소프트포인터)만 수집 → `RequestAsyncLoad`
- 리롤 시 이전 `IconLoadHandle`을 `CancelHandle()`로 취소 후 새 로드 시작
- `NativeDestruct`에서도 핸들 취소 (위젯 소멸 시 자원 반환)
- 로드할 아이콘이 없으면 즉시 `PopulateOfferCards()` 호출 (아이콘 미설정 DA 대응)

`NSDataSubsystem::GetData<UNSAugmentDefinition>(Id)` — 인런 진입 시 일괄 로드된 캐시에서 동기 조회. 인런 준비 여부를 `IsRunReady()`로 선확인.

  → UIManagerSubsystem::SelectAugmentCardByIndex(N) / RequestRerollAugment()

      → AugmentationWidget::SelectCardByIndex(N) / RequestRerollAugment()
