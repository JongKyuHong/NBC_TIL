설계 요약 (아웃런 파츠 NPC — 언락/장착)

  1. 데이터 모델 (증강 동형, DA 정체성 유지)
  - 신설 FNSPartDefinitionRow : FTableRowBase (DT_PartDefinition, 카탈로그 주도): PartSlot, bCanReroll, 등급별 ValueRange(DA에서 이사), UnlockCost(신규), TSoftObjectPtr<UNSPartDefinition>
  Definition(DA 참조)
  - UNSPartDefinition 얇게: Icon/PartName/PartMesh/EffectClass/GrantedAbilities만 (PrimaryAsset·번들 유지)
  - 세이브/런타임 정체성 = 기존 TSoftObjectPtr<UNSPartDefinition> 그대로 → 세이브 마이그레이션 없음

  2. 해석 인프라 (증강 패턴 복제)
  - NSDataSubsystem에 DT_PartDefinition 참조 + 로드 시 TMap<FPrimaryAssetId, FNSPartDefinitionRow> 캐시 빌드
  - NSPartUtils에 ResolvePartRow(DefId) 추가 (기존 ResolvePartDefinition과 병행)

  3. 언락 흐름
  - 위젯 → Subsystem::PurchasePart(Def): 비용을 row에서 조회(UI 위변조 방지, 단일 출처) → CommonCurrency 차감 → OwnedParts 추가
  - LoadSynchronous 제거: ValueRange가 row에 있으므로 구매 시 DA 로드 불필요(부수효과로 동기로드 규칙 위반 자동 해소)

  4. 장착 흐름 (저장 전용)
  - 위젯 → Subsystem::SetEquippedPart(CharacterId, Def, Common): 소유 검증 후 캐릭터 슬롯에 참조 저장
  - 허브 즉시 GAS 장착(현 Server_RequestEquip) 경로는 아웃런에서 제거 — 기획서대로 "런 시작 시 적용". (이게 1번 미해결이던 부분 → 저장 전용으로 결정. 혹시 허브 비주얼 프리뷰를
  의도하셨으면 말씀 주세요, 그건 선택적 추가입니다.)

  5. 목록/UI 재연결
  - 위젯의 정적 SelectableParts 배열 폐기 → DT_PartDefinition rows가 언락 가능 목록. 행별로 소유/잠금 상태 + 비용 + 현재 장착 표시. NPC 해금은 NPC 접근 게이팅(IsNPCUnlocked)으로 유지

  6. 런 시작 적용
  - 기존 NSPlayerController 업로드 → EquippedPart 해석 → ApplyEquippedPart() 경로 그대로 동작(변경 없음)

  7. 모든 수치는 DT
  - 언락 비용, 등급별 ValueRange, 리롤 가능 여부, (필요시) 언락 목록 enable 플래그 — 전부 DT_PartDefinition

  확정 등급은 아웃런 Common 고정(인런 등급업은 M3)으로 잡았습니다.

  이 방향이 맞으면 이대로 들고 있겠습니다. 수정하고 싶은 부분 있으면 말씀해 주세요.
