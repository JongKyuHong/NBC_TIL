## 재화 시스템

ANSCurrencyReplicationProxy의 역할이 정확하게 뭐야? 존재는 서버에서 하는건가? 아니면 AActor상속을 받았으니까 복제용인건가?? 얘를 통해서 픽업액터, 재화컴포넌트 이런게 어떤식으로 소통하는거지?

ANSCurrencyReplicationProxy의 역할은 서버에 존재하지만 서버가 특정 한명의 클라이언트에게 명령을 전달하기 위한 통로역할 그래서 Proxy라는 대리자 명칭을 씀
또, Client RPC를 쏘기 위해서 AActor를 사용함, 이 액터는 복제되지만 데이터(프로퍼티)를 복제하려는게 아니고 Client RPC를 쏠 수 있는 자격을 얻기 위해 복제 액터, SceneRoot만 있고 복제 프로퍼티가 하나도 없는게 그 이유

서버에 플레이어 수만큼 생성하고 NSRunGameMode의 PostLogin/Logout에서 생성/소멸 됨
각 클라에는 자기 것 1개만 복제되어 존재, 호스트도 owner 클라 경로로 동일하게 처리

DropSubsystem: 몹 죽음 -> DropId 발급 -> 레지스트리 등록 -> 플레이어의 프록시를 찾아서 ClientRPC 발사(네트워크 경유), 프록시가 드롭 아이디별 픽업을 직접 관리한다.

### FSoftObjectPath, TSoftObjectPtr, TSoftClassPtr

#### 핵심 개념: 하드 참조 vs 소프트 참조

| 구분      | 하드 참조 (`UObject*`, `TObjectPtr`)        | 소프트 참조 (`TSoftObjectPtr`, `TSoftClassPtr`) |
| --------- | ------------------------------------------- | ----------------------------------------------- |
| 로드 시점 | 오브젝트가 생성될 때 **자동으로 같이 로드** | 명시적으로 로드하기 전까지 **메모리에 없음**    |
| 메모리    | 로드 즉시 점유                              | 경로 문자열만 보관                              |
| 사용 시점 | 항상 유효해야 하는 참조                     | 필요할 때 비동기 로드할 에셋                    |
| 위험      | 순환 의존, 메모리 낭비                      | `.Get()`이 null일 수 있음 (로드 전)             |

**중요:** UPROPERTY에 하드 참조를 걸면 그 클래스를 로드할 때 참조된 에셋 전체가 딸려 로드된다. DataAsset 하나가 맵 전체 에셋을 끌고 오는 참사의 원인.

---

#### FSoftObjectPath

```cpp
FSoftObjectPath Path = TEXT("/Game/Assets/SK_Mesh.SK_Mesh");
```

- **순수 경로 문자열.** 타입 정보 없음
- 직렬화, UI 표시, 에셋 레지스트리 조회 등 "경로를 다루는" 로우레벨 작업에 사용
- `RequestAsyncLoad`에 직접 넣을 수 있음
- Blueprint 노출보다는 C++ 내부 로직에서 경로 조작 시 사용

#### TSoftObjectPtr\<T>

```cpp
TSoftObjectPtr<UStaticMesh> Mesh;
```

- **타입이 있는 소프트 참조.** `FSoftObjectPath`를 감싼 래퍼
- 에디터에서 드래그&드랍으로 에셋 지정 가능 (타입 필터링 지원)
- `.Get()` → 이미 로드되어 있으면 포인터 반환, 없으면 `nullptr`
- `.IsValid()` → 로드되어 있고 유효한지
- `.IsNull()` → 경로 자체가 비어 있는지
- UPROPERTY에 걸어도 참조한 에셋을 자동 로드하지 않음 → **메모리 안전**
- 사용 전 `RequestAsyncLoad` 또는 `LoadSynchronous` 필요

```cpp
// 비동기 로드 (권장)
StreamableManager.RequestAsyncLoad(Mesh.ToSoftObjectPath(), [this]()
{
    UStaticMesh* Loaded = Mesh.Get(); // 이제 유효
});

// 동기 로드 (게임스레드 블로킹 — 금지 원칙)
UStaticMesh* Loaded = Mesh.LoadSynchronous();
```

#### TSoftClassPtr\<T>

```cpp
TSoftClassPtr<UGameplayEffect> EffectClass;
```

- **클래스(UClass) 전용 소프트 참조.** `TSoftObjectPtr<UClass>`와 개념적으로 동일하지만 타입 안전성 강화
- `TSubclassOf<T>` (하드)의 소프트 버전
- `.Get()` → `TSubclassOf<T>` 반환 (로드된 경우)
- GE/GA 같은 Blueprint 파생 클래스를 DataAsset에서 소프트 참조로 보관할 때 표준 패턴

```cpp
// 패턴: DataAsset에 선언 → 필요할 때 비동기 로드 → SpawnActor/ApplyGE
TSoftClassPtr<UGameplayEffect> EffectClass;

// 로드 후 사용
TSubclassOf<UGameplayEffect> LoadedClass = EffectClass.Get();
if (LoadedClass)
{
    ASC->ApplyGameplayEffectToSelf(LoadedClass->GetDefaultObject<UGameplayEffect>(), 1.f, EffectContext);
}
```

---

#### 정리: 언제 어떤 걸 쓰나

| 상황                                              | 타입                   |
| ------------------------------------------------- | ---------------------- |
| 경로를 문자열로 다뤄야 할 때 (레지스트리, 직렬화) | `FSoftObjectPath`      |
| DataAsset에 에셋(메시, 텍스처, 사운드) 참조       | `TSoftObjectPtr<T>`    |
| DataAsset에 클래스(GE, GA, Widget) 참조           | `TSoftClassPtr<T>`     |
| 반드시 항상 메모리에 있어야 하는 참조             | `TObjectPtr<T>` (하드) |

### DataTable vs DataAsset

#### UDataTable

- **행(Row) 기반 데이터.** 엑셀/CSV로 일괄 편집 가능
- 스키마: `FTableRowBase`를 상속한 USTRUCT
- 용도: **밸런스 수치, 텍스트 데이터, 반복 패턴** (몬스터 스탯, 드랍률, 보상 테이블)
- 편집: 에디터 내 테이블 뷰 or CSV 임포트
- 조회: `FindRow<FMyRow>(FName, Context)`
- 특징
  - 데이터가 많아져도 구조가 단순하게 유지됨
  - 기획자가 코드 없이 CSV로 수치 조정 가능
  - 에셋 참조(메시, 클래스 포인터)를 담는 건 가능하지만 어색함

```cpp
USTRUCT(BlueprintType)
struct FNSEnemyDropRow : public FTableRowBase
{
    GENERATED_BODY()

    UPROPERTY(EditDefaultsOnly)
    float DropChance = 0.3f;

    UPROPERTY(EditDefaultsOnly)
    int32 MinAmount = 10;

    UPROPERTY(EditDefaultsOnly)
    int32 MaxAmount = 50;
};

// 사용
if (const FNSEnemyDropRow* Row = DropTable->FindRow<FNSEnemyDropRow>(EnemyId, TEXT("")))
{
    // Row->DropChance, Row->MinAmount ...
}
```

#### UDataAsset / UPrimaryDataAsset

- **단일 오브젝트 구조.** 한 에셋 = 하나의 개념적 단위
- 용도: **구조적 참조, 설정 묶음** (캐릭터 정의, 파츠 정의, 능력치 셋)
- 특징
  - 에셋 번들(`AssetBundles`) 태그로 그룹 로드 제어 가능
  - `UPrimaryDataAsset`은 에셋 매니저로 ID 기반 조회 가능
  - 메시, 클래스 포인터, 복잡한 중첩 구조체를 담기에 적합
  - 소프트 참조(`TSoftObjectPtr`)와 함께 쓰면 온디맨드 로드 가능

```cpp
UCLASS()
class UNSPartDefinition : public UPrimaryDataAsset
{
    UPROPERTY(EditDefaultsOnly)
    TSoftClassPtr<UGameplayEffect> EffectClass;  // 소프트 참조

    UPROPERTY(EditDefaultsOnly)
    TSoftObjectPtr<USkeletalMesh> PartMesh;       // 소프트 참조
};
```

#### 역할 분리 원칙

```
DataTable  = "이 적이 얼마를 드랍하나?" (수치/밸런스)
DataAsset  = "이 파츠가 어떤 GE 클래스를 쓰나?" (구조/참조)
```

- DataAsset에 float/int 수치가 늘어나기 시작하면 → DataTable로 분리 검토
- DataTable에 메시/클래스 포인터가 많아지면 → DataAsset으로 분리 검토

### 소프트 참조 비동기 로드 패턴 (UE5)

#### FStreamableManager / UAssetManager

```cpp
// 단일 에셋
FStreamableManager& Manager = UAssetManager::GetStreamableManager();
Manager.RequestAsyncLoad(
    SoftPtr.ToSoftObjectPath(),
    FStreamableDelegate::CreateUObject(this, &UMyClass::OnLoaded)
);

// 여러 에셋 묶음
TArray<FSoftObjectPath> Paths = { MeshA.ToSoftObjectPath(), MeshB.ToSoftObjectPath() };
Manager.RequestAsyncLoad(Paths, FStreamableDelegate::CreateLambda([this]()
{
    // 모두 로드 완료
}));
```

#### 로드 핸들 관리

```cpp
TSharedPtr<FStreamableHandle> LoadHandle;

// 저장해두면 중간에 취소 가능
LoadHandle = Manager.RequestAsyncLoad(Path, Callback);

// EndPlay 등에서 취소
if (LoadHandle.IsValid())
{
    LoadHandle->CancelHandle();
    LoadHandle.Reset();
}
```

#### .Get()의 null 체크 패턴

```cpp
// 잘못된 패턴: 로드 여부 무시
UStaticMesh* Mesh = SoftMesh.Get(); // null일 수 있음
SetStaticMesh(Mesh);                // 크래시 위험

// 올바른 패턴: 로드 확인 후 사용
if (UStaticMesh* Mesh = SoftMesh.Get())
{
    SetStaticMesh(Mesh);
}
else
{
    // 비동기 로드 후 재진입
    LoadHandle = Manager.RequestAsyncLoad(SoftMesh.ToSoftObjectPath(), [this]()
    {
        SetStaticMesh(SoftMesh.Get());
    });
}
```
