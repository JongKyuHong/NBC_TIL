# 재화 시스템 정리

## 1. 시스템 개요

몬스터가 죽으면 재화가 드랍되고, 각 플레이어가 독립적으로 줍는 구조.
A가 줘도 B 화면엔 그대로 남는다. 서버 복제 액터가 0개라 스폰 비용이 낮다.

투사체 시스템(`NSProjectileManagerComponent` + `NSProjectileReplicationProxy` + `NSProjectileVisual`)과
동일한 패턴을 그대로 미러링했다. 차이가 있다면 WorldSubsystem을 활용했다.

---

## 2. 구성 요소

| 클래스                        | 위치                          | 역할                                    |
| ----------------------------- | ----------------------------- | --------------------------------------- |
| `UNSCurrencyDropSubsystem`    | WorldSubsystem (서버)         | 드랍 레지스트리. DropId 발급, 픽업 검증 |
| `ANSCurrencyReplicationProxy` | Actor, Owner=PlayerController | 서버→클라 이벤트 전달 전용 (Client RPC) |
| `ANSLocalCurrencyPickup`      | Actor, bReplicates=false      | 클라 로컬 비주얼 + 충돌 감지            |
| `UNSCurrencyComponent`        | PlayerState 컴포넌트          | 임시재화 지갑(Wallet) + 런 버킷         |
| `UNSPlayerProgressComponent`  | PlayerState 컴포넌트          | 영구재화 SoT, 세이브 대상               |
| `UNSCurrencyVisualData`       | DataAsset                     | (Type, Grade) → Mesh 매핑               |

---

## 3. 동작 흐름

### 드랍 생성

```
몬스터 사망
→ NSRunGameMode::NotifyEnemyKilled (서버)
→ UNSCurrencyDropSubsystem::RegisterDrop(Type, Grade, Amount, Location, Duration)
  → DropId 발급, ActiveDrops에 FNSCurrencyDropEntry 등록
  → 등록된 모든 프록시에 SendSpawnEvent(FNSCurrencySpawnEvent)
    → 각 ANSCurrencyReplicationProxy::Client_SpawnCurrency (Client RPC)
      → 클라: ANSLocalCurrencyPickup 스폰 (Deferred + Initialize)
        → 비동기 메시 로드 (UNSCurrencyVisualData 조회)
        → 만료 타이머 시작 (Duration초 후 Destroy)
```

### 픽업

```
로컬 폰이 픽업 액터에 오버랩
→ SetActorHiddenInGame(true) — 즉각 UX (클라)
→ ANSPlayerState::Server_CollectCurrency(DropId) — 서버 RPC
  → UNSCurrencyDropSubsystem::TryCollect(DropId, PlayerState)
    검증 4단계:
    1. DropId 존재 여부
    2. 만료 시간 초과 여부
    3. 이미 이 플레이어가 주운 적 있는지 (CollectedPlayer)
    4. 서버 기준 거리 (300cm²)
    → 통과 시:
      - Currency.Temp → CurrencyComponent::AddTemp → Wallet 가산
      - Currency.Common/Skill → CurrencyComponent::AddRunPermanent → PendingPermanent 버킷
      - CollectedPlayer.Add
      - SendRemoveEvent(DropId) → 해당 클라 픽업 액터 Destroy
    → 실패 시:
      - SendRestoreEvent(DropId) → 픽업 액터 다시 보이게 복원
```

### 런 종료 (영구재화 확정)

```
NSRunGameMode::OnResultDisplayFinished
→ 각 PlayerState::CurrencyComponent::CommitRunPermanent(Multiplier)
  - 클리어: Multiplier = 1.0
  - 전멸: Multiplier = 0.5
  → PendingPermanent × Multiplier
  → Currency.Common → ProgressComponent::AddCommonCurrency
  → Currency.Skill → ProgressComponent::AddJobCurrency
  → PendingPermanent 비움
→ SaveAllPlayersProgress()
  → ProgressComponent::BuildPayload → Client_SaveProgress RPC
  → 클라 로컬 SaveGame 저장
```

## 4. 재화 종류별 저장 구조

| 재화            | 런 중 보관               | 런 종료 후                       | 저장 위치                     |
| --------------- | ------------------------ | -------------------------------- | ----------------------------- |
| Currency.Temp   | Wallet (FastArray, 복제) | 소멸                             | 저장 안 함                    |
| Currency.Common | PendingPermanent 버킷    | ProgressComponent.CommonCurrency | UNSPermanentSaveGame (계정)   |
| Currency.Skill  | PendingPermanent 버킷    | ProgressComponent.JobCurrency    | FNSCharacterSaveData (캐릭터) |

**용어 정리**

- **버킷(PendingPermanent)**: 런 중 영구재화를 임시로 쌓아두는 서버 전용 TMap. 런 종료 시 배율 적용 후 ProgressComponent로 이동하고 비워진다.
- **지갑(Wallet)**: 임시재화 전용. FastArray로 복제돼 HUD에 실시간 표시. 저장 안 됨.
- **영구 지갑**: ProgressComponent. 세이브 대상. Wallet이 아님.

---

## 5. 프록시 생명주기

`ANSRunGameMode`가 관리. 투사체 프록시와 나란히 처리됨.

| 이벤트                     | 동작                                             |
| -------------------------- | ------------------------------------------------ |
| BeginPlay                  | 이미 접속한 PC 전원 EnsureCurrencyProxy          |
| PostLogin                  | 새 PC → EnsureCurrencyProxy                      |
| HandleSeamlessTravelPlayer | 레벨 이동 후 재등록                              |
| Logout                     | DestroyCurrencyProxy → UnregisterProxy → Destroy |

프록시는 `Owner = PlayerController`, `bOnlyRelevantToOwner = true`라 해당 클라에만 복제됨.
서버가 `SendSpawnEvent`를 호출하면 그 PC의 클라에만 Client RPC가 날아감.

# 언리얼엔진

우리는 SeamlessTravel을 사용한다. 다음 스테이지던 거점 복귀든 둘 다 ServerTravel임
레벨이 바뀌면 World도 GameState도 둘다 새로 생성 됨
UE에서 레벨(맵) 전환 = 맵 트레블, 트래블하면 새 UWorld가 로드되고 기존 World는 파괴됨, GameState와 WorlSubsystem은 둘다 World에 종속되어서 같이 죽고 같이 새로 생김
