# 내용 정리

| 항목                     | 역할                                                   |
| ------------------------ | ------------------------------------------------------ |
| DDD (Data-Driven Design) | 하드코딩 대신 DataTable/DataAsset으로 게임 로직 구동   |
| CSV, JSON 익스포팅       | DataTable의 데이터를 외부 파일로 내보내는 기능         |

| 비동기 로드              | StreamableManager로 로딩 중에도 게임 진행              |
| Primary Data Asset       | Asset Manager가 관리할 수 있는 핵심 데이터 에셋 클래스 |

## Hard Reference vs Soft Reference

```c++
UPROPERTY()
USkeletalMesh* Mesh;  ← Hard Reference (하드 레퍼런스)

UPROPERTY()
TSoftObjectPtr<USkeletalMesh> Mesh;  ← Soft Reference (소프트 레퍼런스)
```

- 하드 레퍼런스는 "이 클래스가 로드될 때 이 메시도 무조건 같이 메모리에 올려" 라는 의미
- 소프트 레퍼런스는 "이 메시의 경로는 알고 있는데, 당장 메모리에 올리지는 않아" 라는 의미

### StreamableManager

- 소프트 레퍼런스로 저장된 경로를 보고 실제로 파일을 디스크에서 메모리로 올려주는 주체

## 에셋 매니저

Asset Manager는 GC를 무시하고 강제로 메모리에 고정, 캐릭터 Actor가 Destroy되어도 DA는 메모리에 남아있음

### 왜?? 그렇게 설계

일반 UObject는 참조하는 Actor가 Destroy되면 GC가 수거 가능
Asset Manager로 로드한 에셋은

## AssetManager

StreamableManager를 내부에 포함하면서, 거기에 "추적" 기능이 추가된 상위 개념

## PrimaryDataAsset

AssetManager가 추적할 수 있는 에셋과 그렇지 않은 에셋의 차이
