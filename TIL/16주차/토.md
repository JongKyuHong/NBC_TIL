# 멀티플레이 개념 정리

## NetMode

나는 지금 무엇인가?? 현재 실행중인 게임 인스턴스가 어떤 네트워크 역할인지 나타냄

- StandAlone : 싱글플레이
- ListenServer : 서버이면서 직접 플레이
- DedicatedServer : 플레이없이 서버만
- Client : 서버에 접속한 클라이언트 (일반 플레이어)

## NetDriver

통신 자체를 관리하는 관리자, 실제 패킷 송수신, 연결관리등을 처리

- 서버의 NetDriver : 접속한 클라이언트 수만큼 NetConnection을 보유하고 관리
- 클라이언트의 NetDriver : 서버로 향하는 NetConnection 딱 1개만 관리

## NetConnection

서버<-> 클라이언트 1:1 통신 경로

- 개발 연결 통로
- 이 통로안에 액터채널(UActorChannel)등이 있다.
- Replicate되는 모든 액터는 이 채널을 통해 데이터를 주고 받는다.

## NetRole

NetMode가 게임인스턴스 전체의 역할이면 NetRole은 개별 액터 하나하나의 역할

- Authority : 서버, 이 액터의 진짜 원본, 상태를 자유롭게 변경가능
- AutonomousProxy : 클라이언트, 내가 직접 조종하는 내 캐릭터 복제본
  - 입력 미리 예측
  - 조금 더 부드럽게 느껴짐
- SimulatedProxy : 클라이언트, 남이 조종하는 다른 캐릭터 복제본
  - 서버 데이터를 보간해서 따라감

### LocalRole vs RemoteRole

- GetLocalRole : 현재 머신에서 이 액터가 가진 역할
- GetRemoteRole : 반대편 머신에서 이 액터가 가진 역할

LocalRole만으로는 서버에서 액터를 구분할 수 없다. 서버입장에서 모든 액터의 LocalRole은 Authority기 때문에 서버에서 스폰된 액터랑 클라이언트에서 접속한 액터랑 구분이 안감 이때 RemoteRole을 사용하면 구분가능

- AActor::HasAuthority() : 서버인지 확인, 게임 로직 처리 권한여부
- AController::IsLocalController() : 현재 내가 조종하는 컨트롤러인지 확인
- APawn::IsLocallyControlled() : 내가 조종하는 Pawn인지 확인

## PRC

내 머신에서 호출하지만 다른 머신에서 실행되는 함수

클라이언트 (공격 버튼 누름) -> ServerRPC호출 -> 서버에서 공격처리

- Server : 클라이언트에서 호출, 서버에서 실행, 클라이언트 -> 서버에 명령전달
- Client : 서버에서 호출, 특정 클라이언트에서 실행, 서버 -> 특정 클라이언트에 명령 전달
- NetMulticast : 서버에서 호출, 서버 + 모든 클라이언트에서 실행 -> 폭발 이펙트, 사운드등 전체 동기화

- Reliable : 반드시 도달 보장(패킷 손실시 재전송) -> 체력감소, 아이템 획득등 중요한 로직, 부하 있음
- Unreliable : 도달 보장 없음(빠르지만 손실가능) -> 이펙트, 사운드등 휘발성 이벤트, 부하 적음

RPC를 호출하는 액터는 반드시 Replicated여야 한다./

### Call vs Invoke

- Call

정적 호출, 컴파일 타임에 어떤 함수인지 어디서 실행될지 이미 결정, 일반적인 함수 호출

- Invoke

동적 호출, 런타임에 어떤 함수인지 어디서 실행될지가 결정, 함수를 간접적으로 실행, 함수 포인터, 동적 바인딩(가상 함수), RPC

- RPC가 Invoke인 이유

RPC를 호출(Call)하는 시점과 실제로 실행(Invoke)되는 시점/장소가 다름
클라이언트에서 ServerRPC 호출(Call) -> 실제 실행(Invoke)는 서버에서 런타임에 결정되어 실행

## 레플리케이션

- 서버가 게임 상태 데이터를 클라이언트에 보내 동기화하는 과정 전체
- 진짜 게임상태는 서버에 존재하고 클라이언트는 이 복제본을 받아 화면에 렌더링

### Property Replication

- 변수 값이 변하면 자동으로 서버 -> 클라이언트에 동기화 되는 방식
- 필요한 변수에만 등록해서 사용 가능
- RepNotify(ReplicatedUsing)을 사용하면 값이 변하는 순간 특정 함수를 자동으로 실행할 수 있다.

```c++
UPROPERTY(ReplicatedUsing = OnRep_HP)
float HP;

UFUNCTION()
void OnRep_HP();
```

### RPC

특정 함수를 다른 머신에서 실행시키는 방식, 값이 아니라 `동작`을 전달할 때 사용

# 궁금한거

## 1. ENGINE_API

- ENGINE_API는 이 함수/클래스를 모듈 DLL밖에서도 쓸 수 있도록 공개한다는 표시
  - export같은 개념

언리얼은 코드를 모듈 단위로 쪼개서 각각 DLL파일로 컴파일한다. 그런데 DLL은 기본적으로 내부 심볼을 외부에서 접근하지 못하도록 막혀있어서 다른 모듈에서 쓰고싶은 함수나 클래스는 명시적으로 내보내기표시를 해주어야 한다. 그게 `ENGINE_API`가 하는 역할
명명규칙이 모듈이름\_API이고 프로젝트이름\_API도 이런느낌

## 2. .lib와 dll차이

.lib는 컴파일 타임에 코드를 실행파일안에 박아넣고, .dll은 런타임에 따로 불러오는 방식이다.

### 정적 라이브러리(lib)

빌드할때 링커가 .lib파일 안의 코드를 .exe안에 통째로 복사해넣는다. 그래서 .exe 하나만 있으면 어디서든 실행되지만 실행 파일 크기가 커지고 같은 코드를 쓰는 프로그램이 여러 개 있으면 메모리에도 여러 번 올라간다.
라이브러리 코드가 바뀌면 **다시 빌드해야 반영**된다.

### 동적 라이브러리(.dll)

.dll은 실행파일안에 코드를 넣지 않고, 이름만 기억해두었다가 **프로그램이 실제로 실행될 때 OS가 메모리에 로드**한다.
여러 프로그램이 같은 .dll을 공유해서 쓸 수 있어서 메모리 효율이 좋고, .dll만 교체하면 재빌드 없이 버그 수정이나 업데이트가 가능하다. 대신 실행 환경에 해당 .dll파일이 반드시 존재해야 한다.없으면 프로그램 바로 죽음

### 왜 .dll쓸때 .lib는 같이 있는가?

.dll배포시에도 .lib가 생기는데 이 .lib는 정적 라이브러리가 아니라 Import Library, 즉 .dll안에 어떤 함수들이 있는지 링커한테 알려주는 안내서 역할이다. 실제 코드는 .dll에 있고, .lib는 빌드 타임에만 쓰이고 배포할 때는 필요 없다.
언리얼이 모듈마다 .dll로 쪼개는 이유도 이것 때문이다. 엔진 전체를 거대한 .exe파일에 박아넣는 대신 모듈별로 .dll을 분리하면 특정 모듈만 교체하거나 에디터/게임빌드를 따로 관리하기 훨씬 편하다.

# 코드카타

## 다리놓기

```c++
// https://www.acmicpc.net/problem/1010

#include <iostream>

using namespace std;

int dp[30][30];

int comb(int n, int r)
{
    if (n == r || r == 0) return dp[n][r] = 1;
    if (dp[n][r] > 0) return dp[n][r];
    return dp[n][r] = comb(n-1, r-1) + comb(n-1, r);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int T;
    cin >> T;
    long long answer = 1;

    for (int i = 0; i < T; i++)
    {
        int N, M;
        cin >> N >> M;

        cout << comb(M, N) << "\n";
    }

    return 0;
}
```

- 파스칼 삼각형에 의해서 n+1Cr+1 = nCr + nCr+1 이다. 그래서 이걸 이용해서 재귀적으로 코드를 짜면 메모이제이션까지 적용해서 풀 수 있다.
- 또, nCn = nC0 = 1 이라는 성질까지 기억해서 풀자
