## Replication Notify

- 클라이언트에 속성값이 레플리케이션 될 때마다 호출되는 함수가 있음
- UPROPERTY(ReplicatedUsing = OnRep_ServerRotationYaw) 이런식으로 UPROPERTY로 변수에 지정해주면됨
- 함수에는 UFUNCTION()이 반드시 붙어야 하고 접두사 OnRep을 붙여주어야 한다.

Replication Notify의 특징은 클라이언트에서만 실행된다. 서버에서도 해당 로직이 수행되어야하면 명시적으로 호출을 따로 해주어야 한다.
블루프린트의 RepNottify와 C++의 OnRep\_함수에도 차이가 있는데

- C++은 클라이언트에서만 호출되고, 명시적으로 호출가능, 속성값이 변할때 호출되고
- 블루프린트에서는 서버와 클라이언트에서 모두 호출되고 명시적으로 호출이 불가, 서버는 항시 호출되고 클라이언트는 속성값이 변할때만 호출된다는 특징이 있다.

## NetUpdateFrequency

- 액터 레플리케이션 빈도의 최대치
- 1초당 몇번 레플리케이션을 시도할지 지정한 값
- 기본값은 100
- 이론적으로 서버는 `1/100`초 간격으로 레플리케이션을 시도
  - 이는 최대치일뿐 이를 보장하지 않음
  - 서버에 성능을 따라감 그래서 그래픽 기능이 없는 데티케이티드 서버가 더 좋은 성능을 발휘
- NetUpdateFrequency 게터, 세터로 둘다
- 처음부터 최적화를 신경쓰지 말자

## 네트워크 부하 줄이는법

- 어떤 액터의 서버 위치와 클라이언트 위치가 동기화되어야 한다했을때

1. 매 틱마다 액터의 서버 위치를 클라이언트 측으로 레플리케이션해서 동기화
2. 액터의 처음 서버 위치와 속도 값을 안다면, 이를 이용해서 예측을 통해 동기화

- 게임모드의 StartPlay()가 호출되지 않으면 액터들의 BeginPlay()가 호출되지 않았음
- 클라이언트에는 게임모드 액터가 복제되지 않는데 이게 어떻게 가능한거지??
- 게임모드는 게임 시작을 본인이 직접하지 않음, 게임 스테이트에게 명령을 내리고 게임 스테이트 액터는 각 클라이언트로 복제되기 때문에 월드에 존재하는 모든 액터에게 BeginPlay() 호출하도록 시킴

## Relevancy

- 연관이 있는 액터에만 레플리케이션 해줌

### 액터 클래스의 연관성 판단 기준

1. Owner : 해당 액터를 소유하고 있는 액터, 해당 액터와 Owner는 연관성이 있다고 판단.
2. Instigator : 해당 액터에게 영향을 끼친 폰, 해당 액터와 Instigator는 연관성이 있다고 판단
3. AlwaysRelevant : 해당 액터가 모든 클라이언트에 항상 연관이 있게끔 설정, ex) 보스몬스터, 레벨내의 모든 플레이어에게 레플리케이션
4. NetUseOwnerRelevancy : 오너 액터의 연관성으로 해당 액터 대신할때 사용, ex) 캐릭터에 달려있는 무기, 캐릭터가 안보이는데 무기가 보일수는 없음
5. OnlyRelevantToOwner : 오너 액터에게만 연관성을 가짐, 다른 액터와는 연관성이 없다. ex) 길라잡이 액터, 다른 플레이어가 굳이 내 길라잡이 액터를 볼필요가 없다.
6. NetCullDistance : 뷰어와의 거리에 따라 연관성 여부를 결정, ex)나뭇잎 액터, 가까우면 레플리케이션, 멀면 안하게끔

### 폰 클래스의 연관성 판단 기준

1. Viewer : 클라이언트 커넥션이 소유하고 있는 플레이어 컨트롤러 가리킴
2. ViewTarget : 플레이어 컨트롤러가 빙의한 폰, Viewer에 빙의한 ViewTarget이 레벨을 돌아다니는 것

# 코드카타

## 풍선 터뜨리기

```c++
// https://www.acmicpc.net/problem/2346

#include <iostream>
#include <vector>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    int N;
    cin >> N;

    vector<pair<int, int>> vec(N);

    for (int i = 0; i < N; i++)
    {
        int tmp;
        cin >> tmp;
        vec[i].first = tmp;
        vec[i].second = i+1;
    }
    int index = 0;

    while (true)
    {
        cout << vec[index].second << " ";
        int current = vec[index].first;
        vec.erase(vec.begin() + index);
        if (vec.empty()) break;
        int size = (int)vec.size();
        if (current > 0)
        {
            index = (index+current-1) % size;
        } else
        {
            index = ((index+current) % size + size) % size;
        }
    }

    return 0;
}
```

- 마지막에 size를 vec.size()로 썼는데 vec.size()는 형이 size_t이다.
- size_t는 음수 int와 %연산을 하면 int가 강제로 size_t로 변환된다고 한다.
  - 여기서 버그가 발생해서 올바르지 않은값이 반환되었다.
- (int)로 명시적으로 형변환을 시켜주니까 정상적으로 정답이 출력되었다.
