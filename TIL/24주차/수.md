# 면접 질문

## 컨테이너 - map 설명 + 시간복잡도에 대해 설명

- 우선 맵은 레드블랙트리 기반으로 구현된 정렬된 레드 블랙 트리이므로 삽입,삭제,탐색은 모두 O(log N)
- 데이터가 늘어도 성증 저하가 일정하게 억제
- 키를 기준으로 오름차순 정렬이 기본값
- upper_bound, lower_bound같은 범위탐색이 가능하다.

### map, unordered_map

- 우선 크게봐서는 정렬이 있고 없고 차이
- unordered_map은 해시테이블로 구성되어있다.
- 탐색, 삽입, 삭제 시간이 평균 O(1), 최악 O(N)
- 순서가 중요하거나 범위 탐색시에는 map, 데이터가 많고 빠른 검색이 최우선일때는 unordered_map을 고려해볼만함

### 레드 블랙 트리, 균형 이진 탐색 트리

- 레드블랙트리는 균형 이진탐색트리의 한 종류
- 다른 균형이진탐색트리인 AVL트리와 비교하면 약간의 불균형을 허용하는 편
- AVL은 탐색속도가 가장 빠르고, 엄격하게보기 때문에 삽입/삭제 시 균형을 맞추기 위해 트리를 재배치(회전)하는 작업이 더 빈번, 레드블랙트리는 회전 연산이 적어 실전 성능에서 유리
- C++의 STL이 레드블랙트리를 선택한 이유는 탐색과 수정사이의 효율적인 균형때문

### 해시 충돌이 잦으면 unordered_map 성능 변화

- um은 해시함수로 키를 버킷 번호로 변환함, 만약 서로 다른 키가 같은 버킷에 배정되면 해시 충돌이 발생
- 충돌이 잦아질경우 시간복잡도 악화, 데이터 개수가 버킷 개수를 초과하면 버킷 크기를 늘리고 전체 데이터를 다시 재배치(Rehash) 이 과정에서 일시적인 성능 저하가 발생함

## 해시 테이블의 동작 원리를 설명하세요

- 무한에 가까운 키 공간을 유한한 배열의 인덱스로 매핑하여 탐색, 삽입, 삭제를 평균 O(1) 시간 내에 처리하는 효율적인 자료구조
- 동작 원리

1. 해시 함수를 통한 해싱
   사용자가 입력한 임의의 길이의 키(문자열, 숫자, 객체등)를 고정된 길이의 정수값인 해시코드(Hash Code)로 변환합니다. 이때 일관성, 균일성이 보장 일관성이란 동일한 입력에는 항상 동일한 출력이 나와야 함, 균일성이란 해시 값이 전체 범위에 골고루 퍼져야 성능이 유지

2. 인덱스 매핑
   해시 코드는 배열의 크기보다 훨씬 큰 숫자일 수 있다. 따라ㅓㅅ 배열의 실제 인덱스로 변환하는 과정이 필요
   보통은 나머지 연산(모듈로)을 사용

3. 버킷 저장
   매핑된 인덱스 위치에 실제 데이터를 저장

### 해시 충돌과 해결 전략

해시 함수가 아무리 휼륭해도 서로 다른키가 동일한 인덱스에 매핑되는 해시 충돌을 피할 수는 없다.
이를 해결하는 방식에 따라 해시 테이블의 성능과 성격이 결정된다.

1. 분리 연결법
   버킷마다 연결리스트나 트리를 두어 충돌된 데이터를 줄줄이 연결하는 방식
   작동 : 충돌이 나면 인덱스의 리스트 뒤에 노드를 추가함
   특징 : 테이블 확장이 덜 빈번하며, 구현이 상대적으로 단순
   고도화 : 자바의 HashMap처럼 노드 개수가 일정 수준을(보통 8개) 넘어가면 리스트를 레드블랙트리로 변경하여 최악의 경우에도 O(log N)의 성능을 보장

2. 개방 주소법
   충돌이 발생하면 비어있는 다른 버킷을 찾아 데이터를 저장하는 방식
   선형 조사 : 다음 칸, 그 다음 칸을 순차적으로 확인, 캐시 효율은 좋으나 데이터 뭉침현상인 클러스터링에 취약
   이차 조사 : 1^1, 2^2, 3^2 칸씩 건너뛰며 빈곳을 찾음
   이중 해싱 : 두번째 해시함수를 사용하여 건너뛸 간격을 결정, 클러스터링 방지에 가장 효과적

### 동적 유지보수 , 부하율과 리해싱

해시 테이블은 공간이 찰수록 성능이 기하급수적으로 떨어짐, 이를 관리하기 위해 부하율 개념을 도입
리해싱 : 부하율이 특정 임계치 (보통 0.7 ~ 0.75)를 넘어서면 배열의 크기를 약 2배로 늘리고 모든 데이터를 새로운 해시값으로 다시 계산하여 배치
이 과정은 O(N)의 비용이 들지만 가끔 발생하는 비용이므로 전체적으로는 O(1)을 유지하게 해줌

### 결론

해시 테이블을 설계할때는

1. 공간과 시간의 트레이드 오프 : 배열 크기를 크게잡으면 충돌은 줄지만 메모리 낭비가 심함
2. 데이터의 순서 : 해시 테이블은 데이터가 입력순서나 키 순서대로 저장되지 않음, 순서가 중요하다면 std::map과 같은 트리 기반 구조를 사용해야 함
3. 해시 Dos 공격 : 공격자가 의도적으로 충돌을 유발하는 데이터를 주입해 서버 성능을 마비시킬 수 있음 이를 막기 위해 현대적 구현체는 해시 함수에 시드를 섞기도 함

효율적인 해시 함수와 현명한 충돌 해결 전략, 적절한 시점의 리해싱을 통해 상수 시간 내에 데이터에 접근할 수 있게 설계된 최적화의 산물, 데이터 구조를 설계할 때 빠른탐색이 가장 큰 목표라면 해시 테이블은 고려해볼만 하다.

## 유니크 값 구하기

- unique로 구할수 있긴한데 이게 연속된 같은 값만 지울 수 있어서 정렬이 필수임

```c++
int main()
{
    vector<int> v = {1, 4, 4, 2, 2, 2, 5, 1};
    sort(v.begin(), v.end());

    auto newEnd = unique(v.begin(), v.end());
    for (int x : v) cout << x << " "; // 1 2 4 5 ? ? ? ?
    v.erase(newEnd, v.end());

    for (int x : v) cout << x << " "; // 1 2 4 5
}
```

- unique는 연속된 중복원소를 제거하고 그 끝위치 iterator를 반환함
- unique는 동작할때

1. 첫번째 1을 보고 유지
2. 두번째 1을 보고 중복임을 확인하고 그 다음 비 중복원소 2를 복사함 -> {1, 2, 2, 2, 2, 4, 4, 5}
3. 첫번째 2를 보고 유지
4. 두번째, 세번째 2는 중복이므로 중복 첫 시작 자리에 다음 비중복 4를 넣음 -> {1, 2, 4, 2, 2, 4, 4, 5}
5. 첫번째 4를 보고 유지
6. 두번째 4는 중복 그 자리에 비중복인 5 넣음 -> {1, 2, 4, 5, 2, 4, 4, 5}
7. 5를 보고 유지 -> 최종상태 -> {1, 2, 4, 5, ?, ?, ?, ?} -> ?는 유효하지 않은값

그래서 항상 erase랑 세트로 사용해서 깔끔하게 중복제거 하자

## 빠른 이진 탐색

- lower_bound, upper_bound는 정렬된 구간에서의 이진 탐색을 깔끔하게 제공

```c++
int main()
{
    vector<int> v = {1, 2, 4, 4, 5, 7, 8};
    auto lb = lower_bound(v.begin(), v.end(), 4);
    auto ub = upper_bound(v.begin(), v.end(), 4);
    cout << (lb - v.begin()) << endl;
    cout << (ub - v.begin()) << endl;
}
```

- lower_bound는 3번째인자 x값 이상이 나오는 첫번째 인덱스
- upper_bound는 3번째인자 x 초과의 값이 처음 나오는 인덱스

## 원소 변환하기

- transform : 구간 내의 원소들을 내가 원하는대로 변환 시켜주는 역할
- 단항 연산 : 각 원소들을 하나씩 변환
- 이상 연산 : 두 구간의 원소들을 짝지어서 변환

1. 단항 연산

```c++
int main()
{
    vector<int> v1 = {1, 2, 3, 4, 5};
    vector<int> v2(5);

    transform(v1.begin(), v1.end(), v2.begin(), [](int x)
    {
        return pow(x, 2); // 각 원소 제곱
    });

    for (int x : v2) cout << x << " "; // 1 4 9 16 25
}
```

- 변환 함수자리에는 미리 만들어둔 함수 객체나 일반 함수를 사용할 수도 있다.

2. 이항 연산

```c++
int main()
{
    vector<int> v1 = {1, 2, 3, 4, 5};
    vector<int> v2 = {6, 7, 8, 9, 10};
    vector<int> v3(5);

    transform(v1.begin(), v1.end(), v2.begin(), v3.begin(), plus<int>());
    for (int x : v3) cout << x << " "; // 7 9 11 13 15
}
```

- v1과 v2의 같은 위치의 원소끼리 더해서 v3에 더하는 예시
- v1, v2는 같은 크기여야 함

## Q1. N개의 정수로 이루어진 정렬된 수열이 주어집니다. 이어서 Q개의 쿼리가 주어지는데, 각 쿼리는 두 정수 A, B로 이루어져 있습니다. 각 쿼리마다 수열에서 A 이상 B 이하인 원소의 개수를 출력하세요. (15분)

```c++
#include <iostream>
#include <algorithm>
#include <vector>

using namespace std;

int main()
{
   int N = 0;
   cin >> N;
   vector<int> v(N);
   for (int i = 0; i < N; ++i) cin >> v[i];

   sort(v.begin(), v.end());

   int Q = 0;
   cin >> Q;
   for (int i = 0; i < Q; ++i)
   {
      int A, B;
      cin >> A >> B;
      auto it1 = lower_bound(v.begin(), v.end(), A);
      auto it2 = upper_bound(v.begin(), v.end(), B);
      cout << it2 - it1 << "\n";
   }
}
```

## Q2. N개의 정수로 이루어진 수열이 주어집니다. 이때, 연속된 K개의 수를 선택했을 때 그 합이 최대가 되는 값을 구하세요. (20분)

```c++
#include <iostream>
#include <algorithm>
#include <vector>

using namespace std;

int main()
{
   int N, K;
   cin >> N >> K;
   vector<int> v(N);
   for (int i = 0; i < N; ++i) cin >> v[i];

   int start = 0;
   int end = 0;
   int sumV = 0;
   int maxV = -100 * 100000;
   while (end < N)
   {
      while (end - start < K)
      {
         sumV += v[end++];
      }
      maxV = max(maxV, sumV);
      sumV -= v[start++];
   }
   cout << maxV;
}
```

- 기존 슬라이딩 윈도우 방식 partial_sum을 이용해서도 풀어보자

```c++
#include <iostream>
#include <algorithm>
#include <vector>
#include <numeric>

using namespace std;

int main()
{
   int N, K;
   cin >> N >> K;
   vector<int> v(N);
   vector<int> psum(N+1, 0);
   for (int i = 0; i < N; ++i) cin >> v[i];

   partial_sum(v.begin(), v.end(), psum.begin()+1);

   int maxV = -100 * 100000;

   for (int i = K; i < N; ++i) maxV = max(maxV, psum[i] - psum[i-K]);
   cout << maxV;
}
```

- 최대값을 초기화할때 INT_MIN을 사용하면 편리하게 할 수 있다고 한다.
