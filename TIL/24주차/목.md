## 📌 알고리즘

### 최대값 / 최소값 탐색

`max_element`, `min_element`, `minmax_element` 모두 `<algorithm>` 헤더 사용.  
반환값은 **이터레이터**이므로 `*`로 역참조해야 실제 값을 얻을 수 있다.

```cpp
vector<int> v = {4, 1, 6, 3, 8, 2};

auto max_it = max_element(v.begin(), v.end());
auto min_it = min_element(v.begin(), v.end());
cout << *max_it << endl; // 8
cout << *min_it << endl; // 1

// 최대 최솟값 한 번에
auto [min_iter, max_iter] = minmax_element(v.begin(), v.end());
cout << *min_iter << ", " << *max_iter << endl; // 1, 8
```

### 조건을 만족하는 원소 개수

| 함수       | 조건             | 특징             |
| ---------- | ---------------- | ---------------- |
| `count`    | 값 일치          | 단순 비교만 가능 |
| `count_if` | 사용자 정의 조건 | 람다 사용 가능   |

```cpp
vector<int> v = {1, 1, 2, 2, 3, 3};

int cnt      = count(v.begin(), v.end(), 2);                          // 2
int even_cnt = count_if(v.begin(), v.end(), [](int x){ return x % 2 == 0; }); // 4
```

### STL 선택 기준 복습

**시간 복잡도 기준**

- 삽입/삭제/탐색이 빈번 → `map` / `set` 또는 `unordered_map` / `unordered_set`
- 최대/최솟값 빈번 추출 → `priority_queue`
- 앞에서 빼고 뒤로 넣기 → `queue`, `deque`

**정렬 필요성 기준**

- 결과적으로 정렬이 필요 → `map`, `set` 또는 `sort`
- 특정 값의 범위 탐색 (정렬 필요) → `lower_bound`, `upper_bound`
- K번째 원소 → `nth_element`

**중복 제거**

- `unique + erase` 패턴 (정렬 필요)
- 또는 `set` / `unordered_set`

### `std::distance` / `std::advance`

```
distance → 이터레이터 간 거리(인덱스 차이) 반환
advance  → 이터레이터를 N칸 전진
```

```cpp
vector<int> v = {1, 2, 3, 4, 5};
auto it = find(v.begin(), v.end(), 3);
cout << *it << endl;              // 3

int idx = distance(v.begin(), it); // 2
advance(it, idx);                  // 2칸 전진
cout << *it << endl;              // 5
```

### 실전 문제 — 중복 제거 후 오름차순 정렬

> N개의 정수에서 중복을 제거하고 오름차순 출력. N ≤ 100,000, 값 ≤ 1,000,000

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;
    vector<int> vec(N);
    for (int i = 0; i < N; ++i) cin >> vec[i];

    sort(vec.begin(), vec.end());           // 1. 정렬
    auto it = unique(vec.begin(), vec.end()); // 2. 중복 몰기
    vec.erase(it, vec.end());               // 3. 중복 제거

    for (int v : vec) cout << v << " ";
    cout << "\n";
    return 0;
}
```

> **포인트:** `unique`는 중복을 제거하는 게 아니라 **뒤로 몰아넣는** 함수다. 실제 삭제는 `erase`로 해야 한다.

### `string::find` vs `std::find` 차이

> 이름만 같을 뿐, **완전히 다른 함수**다.

| 항목             | `string::find`        | `std::find` (`<algorithm>`)     |
| ---------------- | --------------------- | ------------------------------- |
| 헤더             | `<string>`            | `<algorithm>`                   |
| 종류             | 멤버 함수             | 전역 함수                       |
| 호출 방식        | `s.find("abc")`       | `find(v.begin(), v.end(), val)` |
| 반환값           | **인덱스** (`size_t`) | **이터레이터**                  |
| 실패 시          | `string::npos`        | `end()` 이터레이터              |
| 부분 문자열 검색 | ✅                    | ❌ (값 하나만)                  |

```cpp
// string::find
string s = "hello world";
size_t pos = s.find("world");          // 6
size_t pos2 = s.find('o', 5);          // 5번 인덱스부터 탐색
if (s.find("xyz") == string::npos) {} // 실패 체크

// std::find
vector<int> v = {1, 2, 3, 4, 5};
auto it = find(v.begin(), v.end(), 3);
if (it != v.end()) cout << *it;        // 3
```

## 📌 면접 준비 — 멀티스레드

### 멀티스레드 환경이란?

하나의 프로세스 안에서 여러 실행 흐름(스레드)이 동시에 돌아가는 환경.  
각 스레드는 **독립적인 스택**을 갖지만, **힙 메모리와 전역 변수는 모든 스레드가 공유**한다.

---

### 핵심 3대 문제

**① 경쟁 조건 (Race Condition)**

두 스레드가 동시에 같은 변수를 읽고 쓸 때 발생. `counter++`는 읽기 → 더하기 → 쓰기 **3단계**로 쪼개지므로, 그 사이에 다른 스레드가 끼어들면 값이 손실된다.

```cpp
int counter = 0;
// Thread A, B가 동시에 counter++ 실행
// → 2가 되어야 하는데 1이 될 수 있음
```

**② 데드락 (Deadlock)**

스레드 A가 락1을 점유하고 락2를 기다리고, 스레드 B는 락2를 점유하고 락1을 기다리는 상황. **서로가 서로를 영원히 기다린다.**

**③ 메모리 가시성 문제 (Visibility)**

컴파일러와 CPU는 성능을 위해 코드 실행 순서를 최적화(reordering)한다. 스레드 A가 변수를 수정해도 스레드 B의 CPU 캐시에 반영이 안 될 수 있다.

---

### C++ 동기화 도구

| 도구                      | 헤더                   | 용도                             |
| ------------------------- | ---------------------- | -------------------------------- |
| `std::mutex`              | `<mutex>`              | 임계 영역 보호 (가장 기본)       |
| `std::lock_guard`         | `<mutex>`              | RAII 방식 mutex 자동 관리        |
| `std::unique_lock`        | `<mutex>`              | 조건 변수와 함께 사용, 유연한 락 |
| `std::atomic<T>`          | `<atomic>`             | 락 없이 원자적 연산              |
| `std::condition_variable` | `<condition_variable>` | 스레드 대기/알림 (생산자-소비자) |

---

### Mutex란?

**Mutual Exclusion(상호 배제)** 의 약자. 한 번에 딱 하나의 스레드만 공유 자원에 접근하도록 막아주는 **자물쇠**.

```cpp
#include <mutex>

std::mutex mtx;
int shared_data = 0;

// ❌ 직접 lock/unlock — 예외 발생 시 unlock이 안 불림
void thread_func() {
    mtx.lock();
    shared_data++;
    mtx.unlock();
}

// ✅ lock_guard 사용 — 스코프 벗어나면 자동 해제 (RAII 패턴)
void thread_func2() {
    std::lock_guard<std::mutex> lock(mtx);
    shared_data++; // 예외가 터져도 lock이 자동 해제됨
}
```

---

### 임계 구역 (Critical Section)

여러 스레드가 **동시에 실행하면 절대 안 되는 코드 영역**. Mutex로 이 구간을 보호한다.

```cpp
int counter = 0;

void bad_func() {
    counter++; // ← 임계 구역 (보호 없음, 위험)
}

void good_func() {
    std::lock_guard<std::mutex> lock(mtx);
    counter++; // ← 이제 한 번에 하나의 스레드만 실행
}
```

### 멀티스레드에서 안전한 자료구조

#### 1. `std::atomic<T>` — 단순 타입의 원자적 변수

락 없이 CPU 레벨에서 원자성을 보장. `int`, `bool`, 포인터 같은 **단순 타입에만** 적용 가능.

```cpp
#include <atomic>

std::atomic<int> counter = 0;
std::atomic<bool> bIsRunning = true;

counter++;                                          // 원자적 증가
counter.fetch_add(5);                               // 원자적 덧셈
int val = counter.load(std::memory_order_acquire);  // 안전한 읽기
counter.store(10, std::memory_order_release);       // 안전한 쓰기
```

#### 2. Thread-Safe Queue (생산자-소비자 패턴)

게임에서 이벤트 큐, 네트워크 패킷 큐 구현 시 가장 많이 쓰이는 패턴.

- `wait_and_pop` : 데이터가 없으면 생길 때까지 **스레드 대기** (전용 소비자 스레드용)
- `try_pop` : 데이터가 없으면 즉시 빈 값 반환, **블로킹 없음** (메인 루프용)

```cpp
#include <queue>
#include <mutex>
#include <condition_variable>
#include <optional>

template<typename T>
class ThreadSafeQueue {
public:
    void push(T value) {
        std::lock_guard<std::mutex> lock(mutex_);
        queue_.push(std::move(value));
        cv_.notify_one(); // 대기 중인 소비자 깨우기
    }

    T wait_and_pop() {
        std::unique_lock<std::mutex> lock(mutex_);
        cv_.wait(lock, [this]{ return !queue_.empty(); });
        T val = std::move(queue_.front());
        queue_.pop();
        return val;
    }

    std::optional<T> try_pop() {
        std::lock_guard<std::mutex> lock(mutex_);
        if (queue_.empty()) return std::nullopt;
        T val = std::move(queue_.front());
        queue_.pop();
        return val;
    }

private:
    std::queue<T> queue_;
    std::mutex mutex_;
    std::condition_variable cv_;
};
```

#### 3. Thread-Safe Stack

Stack은 조건 변수 없이 mutex만으로 충분.  
`empty()`는 `const` 함수인데 내부에서 mutex를 건드리므로, mutex를 `mutable`로 선언해야 한다.

```cpp
#include <stack>
#include <mutex>
#include <optional>

template<typename T>
class ThreadSafeStack {
public:
    void push(T value) {
        std::lock_guard<std::mutex> lock(mutex_);
        stack_.push(std::move(value));
    }

    std::optional<T> pop() {
        std::lock_guard<std::mutex> lock(mutex_);
        if (stack_.empty()) return std::nullopt;
        T val = std::move(stack_.top());
        stack_.pop();
        return val;
    }

    bool empty() const {
        std::lock_guard<std::mutex> lock(mutex_);
        return stack_.empty();
    }

private:
    std::stack<T> stack_;
    mutable std::mutex mutex_; // const 함수에서도 lock 가능하도록
};
```

### 면접 답변 — 멀티스레드 안전 자료구조

> 멀티스레드 환경에서는 여러 스레드가 힙 메모리와 전역 변수를 공유하기 때문에, 동시에 같은 자원에 접근하면 **경쟁 조건(Race Condition)** 이 발생합니다.
>
> 이를 해결하는 기본 원리가 **Mutex(상호 배제)** 입니다. C++에서는 `std::mutex`와 `lock_guard`를 함께 써서 RAII 방식으로 안전하게 락을 관리합니다.
>
> 이 원리를 자료구조에 적용하면 **Thread-Safe Queue**와 **Thread-Safe Stack**을 만들 수 있습니다. Queue는 `mutex`로 push/pop 임계 구역을 보호하고, `condition_variable`로 데이터가 없을 때 소비자 스레드를 대기시킵니다. Stack은 동일하게 mutex로 감싸고, pop 시 빈 경우를 `std::optional`로 처리합니다.
>
> 단순 카운터나 플래그처럼 단일 값을 공유할 때는 mutex 없이 **`std::atomic<T>`** 를 사용하면 더 가볍고 빠릅니다.

#### ⚠️ 면접 주의사항

- `volatile`은 멀티스레드 문제를 해결하지 못한다 — 잘못된 답변
- Lock-Free/CAS는 모르면 언급하지 않는 게 낫다
- 원리 설명 없이 "mutex 쓰면 됩니다" 한 마디로 끝내면 감점
