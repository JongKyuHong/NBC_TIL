## 개념

### FName

TSet, TMap, TArray를 지정할때 타입이나 키를 int32가 아닌 FName으로 많이 쓰는게 궁금했다.
실제로 언리얼에서는 FName을 많이 선호한다고 한다. 우선 int32는 코드에서만 의미가 있고 사람이 봤을때 어떤 값인지 알 수 없다.

```c++
EquippedPartIds = { 14, 27, 3 }  // 14가 뭔지 모름

EquippedPartIds = { "Part_Helmet_01", "Part_Chest_01" }
```

또, UE에서 FName을 선호하는 이유중 하나는 DataAsset, Blueprint의 이름과 직접 매핑이 된다. "Part_Helmet_01"이라는 FName이 있으면 에디터에서 같은 이름의 에셋을 바로 찾을 수 있다.
성능또한 FName은 내부적으로 전역 해시 테이블에 저장돼서 비교가 int32만큼 빠르다.

### TSet의 내부구조

TSparseArray + 해시테이블 구조

TSparseArray은 빈 슬롯을 허용하는 배열이고, 해시테이블은 뭐 해시값으로 TSparseArray의 인덱스를 매핑해준다.
