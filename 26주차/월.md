# NeoSanctum — 데이터 컴포넌트 구현 명세

## 현재 상태

존재하는 파일:

- `NSGameDataSubsystem.h / .cpp` — 이 파일을 읽어서 AssetType 상수, DataCache 패턴, 로딩 방식을 파악한 뒤 작업 시작

존재하지 않는 파일 (전부 새로 생성):

- GameMode, GameState, PlayerController, 컴포넌트 모두 없음

---

## 목표

`UNSGameDataSubsystem` 을 제거하고, 에셋 로딩 책임을 GameState 컴포넌트로 분산한다.

- 허브 레벨 → `ANSHubGameState` 에 `UNSHubDataComponent` 부착
- 인런 레벨 → `ANSRunGameState` 에 `UNSRunDataComponent` 부착
- `ANSPlayerController` 가 각 컴포넌트의 완료 델리게이트에 바인딩해서 로딩화면 제거

## 1. UNSHubDataComponent

허브 레벨에서 필요한 에셋(HubData, PartData)을 로드하고 보유한다.  
거점 GameState 생성자에서 (예시 : `ANSHubGameState`) `CreateDefaultSubobject` 로 등록한다.

## 2. UNSRunDataComponent

인런 레벨에서 필요한 에셋(MonsterData, AugmentData, PartData)을 로드하고 보유한다.  
인런 GameState 생성자에서 (예시 : `ANSRunGameState`) `CreateDefaultSubobject` 로 등록한다.

## 5. ANSPlayerController

로딩화면 표시/제거 담당. 레벨에 따라 HubDataComponent 또는 RunDataComponent의 완료 델리게이트에 바인딩한다.

### NSPlayerController.h

```cpp
#pragma once

#include "CoreMinimal.h"
#include "GameFramework/PlayerController.h"
#include "NSPlayerController.generated.h"

UCLASS()
class NEOSANCTUM_API ANSPlayerController : public APlayerController
{
    GENERATED_BODY()

protected:
    virtual void BeginPlay() override;

private:
    UFUNCTION()
    void OnLevelDataReady();

    void ShowLoadingScreen();
    void HideLoadingScreen();
};
```

### NSPlayerController.cpp

```cpp
#include "NSPlayerController.h"
#include "NSHubDataComponent.h"
#include "NSRunDataComponent.h"

void ANSPlayerController::BeginPlay()
{
    Super::BeginPlay();

    if (!IsLocalController()) return;

    ShowLoadingScreen();

    // 현재 레벨의 GameState 컴포넌트를 확인 후 바인딩
    if (UNSHubDataComponent* HubComp = UNSHubDataComponent::Get(this))
    {
        if (HubComp->IsReady())
        {
            OnLevelDataReady();
        }
        else
        {
            HubComp->OnHubDataReady.AddDynamic(this, &ThisClass::OnLevelDataReady);
        }
    }
    else if (UNSRunDataComponent* RunComp = UNSRunDataComponent::Get(this))
    {
        if (RunComp->IsReady())
        {
            OnLevelDataReady();
        }
        else
        {
            RunComp->OnRunDataReady.AddDynamic(this, &ThisClass::OnLevelDataReady);
        }
    }
}

void ANSPlayerController::OnLevelDataReady()
{
    HideLoadingScreen();
}

}
```

## 6. NSGameDataSubsystem 삭제

`NSGameDataSubsystem.h / .cpp` 를 삭제한다.  
프로젝트 내에서 `UNSGameDataSubsystem` 참조가 있다면 함께 제거한다.  
현재 프로젝트 초기 단계이므로 참조가 없을 가능성이 높다.

# 알고리즘

```c++
// https://school.programmers.co.kr/learn/courses/30/lessons/42584?language=cpp

#include <string>
#include <vector>
#include <iostream>

using namespace std;

vector<int> solution(vector<int> prices) {
    int N = prices.size();
    vector<int> answer(N, 0);

    vector<pair<int, int>> stack;

    for (int i = 0; i < N; i++){
        if (stack.empty() || stack[stack.size()-1].second < prices[i]){
            stack.push_back({i, prices[i]});
        } else {
            while (!stack.empty() && stack[stack.size()-1].second > prices[i]){
                answer[stack[stack.size()-1].first] = i - stack[stack.size()-1].first;
                stack.pop_back();
            }
            stack.push_back({i, prices[i]});
        }
    }

    while (!stack.empty()){
        answer[stack[stack.size()-1].first] = N - 1 - stack[stack.size()-1].first;
        stack.pop_back();
    }
    return answer;
}
```
