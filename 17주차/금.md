## 컴퓨터 구조와 언리얼 엔진

> 언리얼 개발자라면 컴퓨터 구조를 알아야 한다. 하드웨어의 캐시메모리, VRAM 용량을 고려하여 텍스처 스트리밍 수치를 조절할 수 있어야 하고, 멀티코어 프로세싱의 원리를 알면 훨씬 적은 비용으로 더 많은 동시 접속자를 수용할 수 있다.

---

## 컴퓨터가 이해하는 정보

언리얼 엔진도 결국 0과 1의 거대한 흐름이다. 하드웨어 입장에서 중요한 두 가지 종류가 있다.

### 데이터

- 우리가 다루는 수많은 에셋
- 그저 정적인 0과 1의 덩어리, 컴퓨터가 이해하는 데이터

### 명령어

- 데이터 그 자체로는 그냥 정보 덩어리
- 이 데이터를 움직이고 화면에 뿌리게 만드는 실질적인 주체
- 블루프린트, C++ 코드가 명령어에 해당

### 컴퓨터의 4가지 핵심 부품

언리얼 기준에서 4가지 핵심 부품: **CPU, 메모리, 보조기억장치, 입출력장치**

---

## 데이터와 명령어

### 정보의 최소 단위

비트(Bit)와 n비트의 법칙. 전구로 비유하면:

| 비트  | 표현 가능한 정보 수 |
| ----- | ------------------- |
| 1비트 | 2가지 (켜짐/꺼짐)   |
| 2비트 | 4가지               |
| 3비트 | 8가지               |
| n비트 | 2^n가지             |

> 💡 언리얼에서 `bool` 변수는 논리적으로는 1비트(true/false)의 의미를 가지지만, 실제 메모리 레이아웃에서는 하드웨어 처리 효율을 위해 바이트(Byte) 단위로 저장되는 경우가 많다. 이런 사소한 차이가 대규모 프로젝트의 최적화 향방을 결정한다.

---

### 문자 집합과 인코딩

게임 내 대사와 UI는 어떻게 0과 1로 변할까?

- **캐릭터셋(문자집합)**: 컴퓨터가 인식할 수 있는 문자 모음 (ASCII 같은 것)
- **유니코드**: 많은 언어, 이모지까지 모두 집합처럼 만들어 놓은 현대 언리얼 개발의 표준
- **문자 인코딩**: 컴퓨터가 이해하게끔 사람의 문자를 0과 1로 변환
- **문자 디코딩**: 0과 1을 사람이 이해하게끔 문자열로 변환

> ⚠️ 인코딩이 UTF-8로 되었는데 다른 방식으로 디코딩할 때 한글 깨짐 현상이 발생한다.

---

### FString vs FText vs FName

| 타입      | 특징                                                                             | 사용 상황              |
| --------- | -------------------------------------------------------------------------------- | ---------------------- |
| `FString` | 가장 기본적인 문자열, 편집이 자유롭고 내부적으로 인코딩/디코딩이 활발하게 일어남 | 일반 문자열 처리       |
| `FText`   | 로컬라이징(다국어 지원)의 핵심, 문자집합과 인코딩 문제를 엔진 차원에서 관리      | UI 텍스트, 다국어 지원 |
| `FName`   | 문자열을 고유한 ID로 인코딩하여 저장, 이름표로서의 성능 최적화에 특화            | 에셋 이름, 식별자      |

> 💡 외부 데이터 가져올 때(CSV, JSON) 깨진다면 파일의 인코딩 방식과 엔진의 디코딩 설정이 다른 것이다.

---

### 에셋 인코딩

- `.png`, `.fbx`는 엔진에 의해 최적화된 내부 포맷(`.uasset`)으로 인코딩되어 저장됨
- 텍스처: 고품질은 2^16, 일반은 2^8로 비트 법칙이 적용되어 밝기 표현, BC7, ASTC 같은 방식으로 인코딩하여 VRAM에 저장
- 메시: 정점의 위치 정보, 실수형 데이터로 인코딩
- **디코딩**: 하드웨어가 화면에 그림을 그리고 물리 연산을 하는 찰나의 순간에 발생

---

### 소스코드와 명령어

- **고급 언어**: 우리가 쓰는 C++ 같은 것
- **저급 언어**: 컴퓨터가 이해하는 언어
  - **기계어**: 0과 1로 나열된 명령어
  - **어셈블리어**: 기계어를 사람이 어느 정도 이해할 수 있게 대응한 언어

> 게임 개발자는 하드웨어와 밀접하기 때문에 저급 언어가 관찰의 대상이다. "왜 이 함수만 호출하면 프레임이 떨어질까?" 를 소스코드 수준에서 분석해야 한다.

---

### 컴파일 언어 vs 인터프리터 언어

| 구분                | 방식                                          | 언리얼 예시 |
| ------------------- | --------------------------------------------- | ----------- |
| **컴파일 언어**     | 소스코드 전체를 훑어 저급 언어로 한 번에 변환 | C++         |
| **인터프리터 언어** | 소스를 한 줄씩 실시간으로 해석하여 진행       | 블루프린트  |

> 💡 블루프린트는 일종의 인터프리터 방식으로 작동하여 C++보다 유연하지만 상대적으로 느리다. 엔진이 제공하는 **블루프린트 네이티브화** 기능은 이 인터프리터 노드들을 컴파일 언어인 C++ 수준으로 변환해 성능을 올리는 기술이다.

---

### 목적 파일과 링킹

컴파일은 0과 1로 된 목적 코드가 담긴 목적 파일(`.obj`)을 만들어낸다. 하지만 이것만으로는 게임이 실행되지 않는다.

`MyGameMode.obj`는 아직 `MyCharacter`가 메모리 어디에 있는지, 엔진의 로그 출력 기능이 어디에 구현되어 있는지 알지 못한다. 이때 흩어진 목적 파일들과 엔진의 거대한 외부 라이브러리들을 하나로 묶어 연결해주는 작업이 **링킹(Linking)**이다.

> 언리얼 엔진은 수만 개의 소스 파일이 있기 때문에 빌드 시간이 굉장히 오래 걸린다.

## 알고리즘

### 병합 정렬 (Merge Sort)

분할 정복 방식으로 배열을 반씩 나눠 정렬 후 병합. 시간복잡도 **O(n log n)** 보장.

````cpp
#include <iostream>
#include <vector>
using namespace std;

void merge(vector<int>& vec, int left, int mid, int right)
{
    vector<int> tmp;
    int i = left, j = mid + 1;
    while (i <= mid && j <= right)
    {
        if (vec[i] <= vec[j]) tmp.push_back(vec[i++]);
        else tmp.push_back(vec[j++]);
    }
    while (i <= mid) tmp.push_back(vec[i++]);
    while (j <= right) tmp.push_back(vec[j++]);
    for (int i = 0; i < right - left + 1; i++)
        vec[i + left] = tmp[i];
}

void mergeSort(vector<int>& vec, int left, int right)
{
    if (left < right)
    {
        int mid = left + (right - left) / 2;
        mergeSort(vec, left, mid);
        mergeSort(vec, mid + 1, right);
        merge(vec, left, mid, right);
    }
}

### 힙 정렬 (Heap Sort)
최대 힙을 구성 후 루트를 반복 추출하여 정렬. 시간복잡도 O(n log n), 추가 메모리 불필요.

```cpp
#include <iostream>
#include <vector>
using namespace std;

void heapify(vector<int>& vec, int n, int i)
{
    int parent = i;
    int left = parent * 2 + 1;
    int right = parent * 2 + 2;
    if (left < n && vec[left] > vec[parent]) parent = left;
    if (right < n && vec[right] > vec[parent]) parent = right;
    if (parent != i)
    {
        swap(vec[i], vec[parent]);
        heapify(vec, n, parent);
    }
}

void heapSort(vector<int>& vec, int n)
{
    for (int i = n / 2 - 1; i >= 0; i--)
        heapify(vec, n, i);
    for (int i = n - 1; i >= 0; i--)
    {
        swap(vec[0], vec[i]);
        heapify(vec, i, 0);
    }
}

int main()
{
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    vector<int> vec = { 4, 10, 3, 5, 1 };
    int n = (int)vec.size();

    heapSort(vec, n);

    for (int i = 0; i < n; i++) cout << vec[i] << " ";
    cout << endl;

    return 0;
}
````

### 문자열 내 마음대로 정렬하기

n번째 인덱스 문자 기준으로 정렬, 같으면 사전순 정렬.

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int idx;

bool compare(const string& a, const string& b)
{
    if (a[idx] != b[idx]) return a[idx] < b[idx];
    return a < b;
}

vector<string> solution(vector<string> strings, int n)
{
    idx = n;
    sort(strings.begin(), strings.end(), compare);
    return strings;
}

int main()
{
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    vector<string> result = solution({"abce", "abcd", "cdx"}, 1);
    for (const string& a : result) cout << a << endl;

    return 0;
}
```

### 가장 큰 수

숫자 배열을 이어 붙여 가장 큰 수를 만드는 문제. 커스텀 comparator로 문자열 연결 순서를 비교.

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <string>
using namespace std;

bool compare(const int& a, const int& b)
{
    string avalue = to_string(a);
    string bvalue = to_string(b);
    return (avalue + bvalue) > (bvalue + avalue);
}

string solution(vector<int> numbers)
{
    sort(numbers.begin(), numbers.end(), compare);
    string tmp = "";
    for (const int& num : numbers) tmp += to_string(num);
    if (tmp[0] == '0') return "0";
    return tmp;
}

int main()
{
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    string result = solution({6, 10, 2});
    cout << result;

    return 0;
}
```

```json

    {
      "Name": "HPBinder",
      "Enabled": true
    }
  ]
}
```

> ⚠️ JSON에서 마지막 항목 뒤에 쉼표(trailing comma)를 붙이면 파싱 오류가 발생한다.
