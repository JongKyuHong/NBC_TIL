# 몬스터 최적화

## EDMonsterSpawner

기존 방식: UPROPERTY로 직접 에셋 참조, 동기 로드.
개선 후: EDGameDataSubsystem::GetData로 캐싱된 데이터 사용

1. GetData로 캐싱된 데이터를 GetMonsterDataAsset에서 반환 기존에 DataAsset을 하드하게 들고있는 방식이 아닌 경로만 들고 있다가
2. 로딩 파이프라인에서 모든 몬스터 DA를 로드하고 GetMonsterDataAsset에서 그 로드된 데이터의 캐시에 접근해서 불러온다.

## AEDMonsterBase

```c++
/**
    * EDGameDataSubsystem에서 이미 해당 데이터들을 로드했으므로
    * .Get()을 통해 동기적으로 가져옴 (이미 메모리에 있어서 부하없음!)
    *
    */
if (USkeletalMesh* SkelMesh = DataAsset->GetMesh().Get())
{
    GetMesh()->SetSkeletalMesh(SkelMesh);
}
else
{
    // 만약에 대비해 동기 로드 시도 1
    USkeletalMesh* LoadedMesh = UEDAssetManager::Get().LoadAssetSync<USkeletalMesh>(DataAsset->GetMesh().ToSoftObjectPath());
    if (LoadedMesh) GetMesh()->SetSkeletalMesh(LoadedMesh);
}

if (UClass* AnimClass = DataAsset->GetAnimInstance().Get())
{
    GetMesh()->SetAnimInstanceClass(AnimClass);
} else
{
    // 만약에 대비해 동기 로드 시도 2
    UClass* LoadedAnim = UEDAssetManager::Get().LoadAssetSync<UClass>(InDataAsset->GetAnimInstance().ToSoftObjectPath());
    if (LoadedAnim) GetMesh()->SetAnimInstanceClass(LoadedAnim);
}
```

- 여기서 동기로 부르는 이유는 게임을 진행할때 우선 캐싱을 해두기때문에 그냥 꺼내오면 되기때문이고 만약에 캐싱되어있지않아서 비동기로 로드한다고 치면 메시없이 돌아다니는 현상이 생기는것보다 그냥 잠깐 멈추더라도 다 메시들고 나오는게 낫지않나해서 이렇게 함
- 로딩파이프라인에 맞추기, 중복로딩을 제거하고 초기화 속도 빠르게

| 구분                  | 수정 전 (Before)                                            | 수정 후 (After)                                                  |
| :-------------------- | :---------------------------------------------------------- | :--------------------------------------------------------------- |
| **에셋 접근 방식**    | `StreamableManager`를 통한 매번 비동기 로드 요청            | 서브시스템 캐시(`DataCache`)에서 즉시 포인터 참조(`.Get()`)      |
| **초기화 시점**       | `BeginPlay` 내 초기화 함수 주석 처리로 동작 불분명          | `BeginPlay`에서 데이터 에셋 유무를 체크하여 확실하게 초기화 수행 |
| **코드 구조**         | `VisualLoadHandle` 관리 및 `OnVisualsLoaded` 콜백 함수 필요 | 핸들과 콜백을 삭제하고 `LoadVisuals` 함수 내에서 즉시 처리       |
| **안정성 (Fallback)** | 로드 완료 전까지 메쉬가 보이지 않는 현상 발생               | 캐시 미스 시 `LoadAssetSync`를 통한 즉시 로딩으로 비주얼 보장    |

# 알고리즘

## dp

팀플끝날때까지는 알고리즘에 많이 신경 못쓸듯??

```c++
// https://www.acmicpc.net/problem/19947

#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main()
{
    ios_base::sync_with_stdio(false);
    cin.tie(0);

    int H, Y;
    cin >> H >> Y;

    int dp[11];
    dp[0] = H;
    dp[1] = (int)(H*1.05);
    dp[2] = (int)(dp[1]*1.05);
    dp[3] = max({(int)(dp[0]*1.2), (int)(dp[2]*1.05)});
    dp[4] = max({(int)(dp[3]*1.05), (int)(dp[1]*1.2)});
    dp[5] = max({(int)(dp[0]*1.35),(int)(dp[4]*1.05), (int)(dp[2]*1.2)});

    for (int i = 6; i <= 10; i++)
    {
        dp[i] = max({(int)(dp[i-1]*1.05), (int)(dp[i-3]*1.2), (int)(dp[i-5]*1.35)});
    }

    cout << dp[Y];

    return 0;
}
```

- https://www.acmicpc.net/problem/19947
- 쉬운 dp문제 이렇게 풀다가 나중에는 for문안에서 i가 3이상일때랑 i가 5이상일때 체크해서 더 깔끔하게 로직변경

```c++
int dp[11];
dp[0] = H;
for (int i = 1; i <= 10; i++)
{
    dp[i] = (int)(dp[i-1]*1.05);

    if (i >= 3)
    {
        dp[i] = max(dp[i], (int)(dp[i-3]*1.2));
    }

    if (i >= 5)
    {
        dp[i] = max(dp[i],(int)(dp[i-5]*1.35));
    }
}
```
