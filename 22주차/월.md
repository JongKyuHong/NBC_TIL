# TIL - UE5 GAS AnimMontage 비동기 로딩 & AssetBundle 시스템 구축

> 날짜: 2026-04-20  
> 키워드: `UE5`, `GAS`, `TSoftObjectPtr`, `AssetBundle`, `AssetManager`, `EDGameDataSubsystem`

---

## 1. 문제 발단

`BP_GA_BasicAttack_Sword`(부모: `UGA_Base`)에서 몽타주가 재생되지 않는 문제 발생.

```
LogAbilitySystem: Warning: UAbilityTask_PlayMontageAndWait called in Ability 
BP_GA_BasicAttack_Hammer_C_0 failed to play montage None
```

---

## 2. 원인 분석 흐름

### GA_Base 구조
```cpp
// GA_Base.h
UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Skill")
EPlayerAnimNameType MontageName = EPlayerAnimNameType::NONE;  // Enum으로 DA 이름 지정

UPROPERTY()
UAnimMontage* PlayerAnimMontage = nullptr;
```

### ActivateAbility 흐름
```
MontageName (Enum)
    → EDGameDataSubsystem::GetData() 로 DA 조회
    → DA 안의 TSoftObjectPtr<UAnimMontage> AnimMontage
    → AnimMontage.Get() → null!  ← 문제 발생 위치
```

### 근본 원인
`EDGameDataSubsystem`이 `PlayerAnimDataAsset` 자체는 캐싱했지만,  
DA 안의 `TSoftObjectPtr<UAnimMontage>` 필드는 별도 로드가 필요함.  
- `DA 로드 ≠ DA 안의 TSoftObjectPtr 로드`
- `.Get()`은 메모리에 올라온 경우에만 유효 → 로드 안 됐으면 `nullptr`

---

## 3. 해결 전략: AssetBundle 시스템 활용

### 핵심 개념
`AssetBundles` 메타 태그를 TSoftObjectPtr 필드에 달아두면,  
`LoadPrimaryAssetsAsync` 호출 시 번들 이름을 함께 넘겨서 TSoft 필드까지 한 번에 로드 가능.

### DA 클래스에 번들 태그 추가
```cpp
// UEDPlayerAnimDataAsset.h
UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Player|Data",
    meta=(AssetBundles="PlayerAnimData"))
TSoftObjectPtr<UAnimMontage> AnimMontage;

// UEDWeaponDataAsset.h
UPROPERTY(EditDefaultsOnly, meta=(AssetBundles="WeaponData"))
TSoftObjectPtr<UStaticMesh> WeaponStaticMesh;
```

### LoadPhase_Player 수정: 2단계 로드
단순히 `LoadPrimaryAssetsAsync`에 번들을 넘기는 것만으로는  
**처음 로드되는 에셋에서 IsPending이 영원히 1**로 유지되는 문제가 발생.  

원인: `LoadPrimaryAssetsAsync`에 번들을 함께 넘기는 건 **이미 로드된 에셋의 번들 상태 변경** 용도.  
처음 로드 시에는 `ChangeBundleStateForPrimaryAssets`를 별도 호출해야 함.

```cpp
void UEDGameDataSubsystem::LoadPhase_Player()
{
    SetPhase(EDataLoadPhase::LoadingPlayer);
    UEDAssetManager& AM = UEDAssetManager::Get();

    TArray<FPrimaryAssetId> AllIds;
    TArray<FPrimaryAssetId> Temp;
    AM.GetPrimaryAssetIdList(PlayerDataAssetType, Temp);     AllIds.Append(Temp); Temp.Reset();
    AM.GetPrimaryAssetIdList(PlayerAnimDataAssetType, Temp); AllIds.Append(Temp); Temp.Reset();
    AM.GetPrimaryAssetIdList(WeaponDataAssetType, Temp);     AllIds.Append(Temp);

    if (AllIds.IsEmpty())
    {
        OnPlayerDataLoaded();
        return;
    }

    // 1단계: 번들 없이 DA 자체만 먼저 로드
    TSharedPtr<FStreamableHandle> Handle = AM.LoadPrimaryAssetsAsync(
        AllIds,
        {},  // 번들 없음
        FStreamableDelegate::CreateLambda([this, AllIds]()
        {
            // 2단계: DA 로드 완료 후 번들 상태 변경으로 TSoft 필드 로드
            UEDAssetManager& AM2 = UEDAssetManager::Get();
            TArray<FName> Bundles = {
                PlayerDataAssetType.GetName(),
                PlayerAnimDataAssetType.GetName(),
                WeaponDataAssetType.GetName()
            };
            AM2.ChangeBundleStateForPrimaryAssets(
                AllIds,
                Bundles,  // 추가할 번들
                {},       // 제거할 번들
                false,
                FStreamableDelegate::CreateUObject(this, &UEDGameDataSubsystem::OnPlayerDataLoaded)
            );
        })
    );

    PhaseHandles.Add(PlayerDataAssetType.GetName(), Handle);
    PhaseHandles.Add(PlayerAnimDataAssetType.GetName(), Handle);
    PhaseHandles.Add(WeaponDataAssetType.GetName(), Handle);
}
```

### GA_Base 복구: .Get()으로 단순화
번들이 정상 작동하면 비동기 콜백 없이 바로 사용 가능.
```cpp
if (IsValid(PlayerAnimData))
{
    PlayerAnimMontage = PlayerAnimData->AnimMontage.Get();  // 이미 메모리에 로드됨
}
```

## 4. EDGameDataSubsystem 공용 비동기 로드 함수

`EDAssetManager`를 활용해 `TSoftObjectPtr` 비동기 로드 로직을 한 곳으로 통일.

### 헤더 추가
```cpp
// EDGameDataSubsystem.h
DECLARE_DELEGATE_OneParam(FOnSoftAssetLoaded, UObject*);

public:
void RequestAsyncLoad(const TSoftObjectPtr<UObject>& SoftRef, FOnSoftAssetLoaded OnLoaded);
```

### 구현
```cpp
void UEDGameDataSubsystem::RequestAsyncLoad(
    const TSoftObjectPtr<UObject>& SoftRef, FOnSoftAssetLoaded OnLoaded)
{
    if (SoftRef.IsNull())   { OnLoaded.ExecuteIfBound(nullptr); return; }
    if (SoftRef.IsValid())  { OnLoaded.ExecuteIfBound(SoftRef.Get()); return; }

    UEDAssetManager::Get().LoadAssetAsync(
        SoftRef.ToSoftObjectPath(),
        FStreamableDelegate::CreateLambda([SoftRef, OnLoaded]()
        {
            OnLoaded.ExecuteIfBound(SoftRef.Get());
        })
    );
}
```

## 5. PreloadSoftAssets 범용 함수

UI, Monster, Player 등 어떤 타입이든 재사용 가능한 TSoft 프리로드 함수.

```cpp
// 헤더
void PreloadSoftAssets(const FPrimaryAssetType& AssetType,
    TFunction<TArray<FSoftObjectPath>(UObject*)> PathCollector);
```

```cpp
// 구현
void UEDGameDataSubsystem::PreloadSoftAssets(
    const FPrimaryAssetType& AssetType,
    TFunction<TArray<FSoftObjectPath>(UObject*)> PathCollector)
{
    TArray<FSoftObjectPath> SoftPaths;
    TArray<FPrimaryAssetId> Ids;
    UEDAssetManager::Get().GetPrimaryAssetIdList(AssetType, Ids);

    for (const FPrimaryAssetId& Id : Ids)
    {
        if (UObject* DA = GetData<UObject>(Id))
            SoftPaths.Append(PathCollector(DA));
    }
    if (SoftPaths.IsEmpty()) return;

    TSharedPtr<FStreamableHandle> Handle = UEDAssetManager::Get().LoadAssetsAsync(
        SoftPaths,
        FStreamableDelegate::CreateLambda([AssetType]()
        {
            UE_LOG(LogTemp, Log, TEXT("[EDGameDataSubsystem] PreloadSoftAssets 완료: %s"),
                *AssetType.ToString());
        })
    );
    PhaseHandles.Add(FName(AssetType.ToString() + TEXT("_Soft")), Handle);
}
```

### 사용 예시
```cpp
PreloadSoftAssets(PlayerAnimDataAssetType, [](UObject* DA) -> TArray<FSoftObjectPath>
{
    TArray<FSoftObjectPath> Paths;
    if (auto* AnimDA = Cast<UEDPlayerAnimDataAsset>(DA))
        if (!AnimDA->AnimMontage.IsNull())
            Paths.Add(AnimDA->AnimMontage.ToSoftObjectPath());
    return Paths;
});
```

## 6. 핵심 트러블슈팅 정리

| 증상 | 원인 | 해결 |
|---|---|---|
| `PlayerAnimMontage = NULL` | DA 안의 TSoft 필드가 로드 안 됨 | AssetBundle 시스템 적용 |
| `IsPending = 1`이 계속 유지 | `LoadPrimaryAssetsAsync`에 번들 동시 지정 시 처음 로드되는 에셋에 적용 안 됨 | DA 로드 후 `ChangeBundleStateForPrimaryAssets` 별도 호출 |
| `LoadSynchronous`는 되는데 `.Get()`은 안 됨 | DA 경로는 있지만 번들 로드가 안 된 상태 | AssetBundle 태그 + 2단계 로드 |
| `DefaultGame.ini` 설정이 중복 오염 | Project Settings에서 수정 시 ini에 중복 기록됨 | 직접 텍스트 에디터로 `-/+` 중복 항목 정리 |
| `AssetBundles` 태그 적용 후 에디터 재시작 필요 | 핫리로드로는 AssetRegistry에 번들 정보 반영 안 됨 | 컴파일 후 에디터 완전 종료 & 재시작 |

## 7. 오늘 얻은 핵심 인사이트

1. **`DA 로드 ≠ DA 안의 TSoft 필드 로드`** — 항상 명심
2. **`LoadPrimaryAssetsAsync` + Bundle은 이미 로드된 에셋용**,  
   처음 로드 시에는 `ChangeBundleStateForPrimaryAssets` 별도 호출 필요
3. **평타 같은 자주 쓰는 에셋은 게임 시작 시 프리로드**해서 GC 핸들로 메모리 유지
4. **`AssetBundles` 태그 = DA 타입 이름과 동일하게 맞추면 관리가 편함**
5. **`DefaultGame.ini`는 Project Settings로만 수정하면 중복이 쌓일 수 있으니 주기적으로 직접 확인**
