# 트러블슈팅: 플레이어 스켈레탈 메시 미적용 문제

## 개요

멀티플레이어 환경에서 게임 시작 시 플레이어 캐릭터의 스켈레탈 메시가 적용되지 않아 캐릭터가 화면에 보이지 않는 현상 발생.

---

## 원인 분석 및 해결 과정

### 1단계 — 델리게이트 바인딩 타입 오류

**문제**

`OnAllDataLoaded`가 `DECLARE_DYNAMIC_MULTICAST_DELEGATE`로 선언되어 있는데 `AddUObject` / `AddUFunction`으로 바인딩 시도 → 컴파일 에러 발생.

**원인**

Dynamic Multicast 델리게이트는 `AddDynamic`만 사용 가능하며, 바인딩 대상 함수에 `UFUNCTION()`이 반드시 있어야 한다.

| 델리게이트 타입                      | 바인딩 방법  | UFUNCTION 필요 여부 |
| ------------------------------------ | ------------ | ------------------- |
| `DECLARE_MULTICAST_DELEGATE`         | `AddUObject` | 불필요              |
| `DECLARE_DYNAMIC_MULTICAST_DELEGATE` | `AddDynamic` | **필수**            |

**해결**

```cpp
// Before
DataSubsystem->OnAllDataLoaded.AddUFunction(this, FName("ApplyTargetMesh")); // ❌

// After — UFUNCTION() 추가 후 AddDynamic 사용
UFUNCTION()
void ApplyTargetMesh();

DataSubsystem->OnAllDataLoaded.AddDynamic(this, &AEDPlayerCharacter::ApplyTargetMesh); // ✅
```

---

### 2단계 — `FPrimaryAssetId` 키 불일치

**문제**

`DataCache`에는 `PlayerData:DA_Basic`으로 저장되어 있는데, 코드에서 `PlayerData:Basic`으로 조회 → `nullptr` 반환.

**원인**

`UEnum::GetDisplayValueAsText()`는 에디터 빌드(`WITH_EDITOR`)에서만 UMETA `DisplayName`을 반환한다.  
런타임(PIE 포함)에서는 DisplayName을 무시하고 **enum 변수명 자체**(`Basic`)를 반환한다.

```cpp
// EPlayerNameType 선언
UENUM(BlueprintType)
enum class EPlayerNameType : uint8
{
    Basic UMETA(DisplayName = "DA_Basic"), // DisplayName은 런타임에서 무시됨
};

// 런타임 결과
UEnum::GetDisplayValueAsText(EPlayerNameType::Basic).ToString();
// → 에디터: "DA_Basic" ✅
// → 런타임(PIE): "Basic"  ❌ → 캐시 키 불일치
```

**해결**

`GetDisplayValueAsText` 대신 enum 값 → 에셋 이름 직접 매핑 함수로 교체.

```cpp
static FName GetPlayerDataAssetName(EPlayerNameType Type)
{
    switch (Type)
    {
        case EPlayerNameType::Basic:     return FName("DA_Basic");
        case EPlayerNameType::Barbarian: return FName("DA_Barbarian");
        case EPlayerNameType::Druid:     return FName("DA_Druid");
        case EPlayerNameType::Nekku:     return FName("DA_Nekku");
        case EPlayerNameType::Elf:       return FName("DA_Elf");
        default: return NAME_None;
    }
}

// 사용 예
FPrimaryAssetId(PlayerSkinCategory, GetPlayerDataAssetName(EPlayerNameType::Basic));
// → PlayerData:DA_Basic ✅
```

---

### 3단계 - 적용

GetPlayerDataAssetName함수를 사용하는것이 정석에 더 가깝지만 우선 기존 GetDisplayValueAsText를 사용하고 ENUM값에 DA*를 붙여주었음 -> 추후에는 displayName을 모두 제거하고 ENUM값에 DA* 붙인거로 해결

## 결과

- 서버와 클라이언트 모두에서 플레이어 스켈레탈 메시 정상 적용

---

## 얻은 교훈

- `UEnum` UMETA `DisplayName`은 에디터에서는 동작하지만 패키징시 동작하지 않음 -> 패키징에서는 메타데이터가 제거되므로 enum 변수명을 반환함
- `DECLARE_DYNAMIC_MULTICAST_DELEGATE`와 `DECLARE_MULTICAST_DELEGATE`의 **바인딩 방식 차이**를 명확히 구분해야 함
- 에디터에서는 동작해도 패키징시
