a. 주방 시뮬레이터

주방에서 손님 주문을 받아서 요리, 각 요리마다 각자 다른 콤보 시스템, 입력을 얼마나 정확하게 하느냐에 따라 콤보 실패, 성공, 대성공이 뜨고 요리결과에도 영향을 끼침, 요리 결과는 콤보에 따라서 실패, 보통, 수작, 걸작으로 구성되어있음

b. 태그 정리

```c++
State.Chef
├── State.Chef.Idle        // 대기 중
├── State.Chef.Cooking     // 요리 중
└── State.Chef.Plating     // 플레이팅 중

Dish                          // 요리 종류
├── Dish.Ramen                // 라면
├── Dish.Stirfry              // 볶음
├── Dish.Sashimi              // 사시미
└── Dish.Soup                 // 찌개

Combo.Judge                   // 콤보 윈도우 입력 판정
├── Combo.Judge.Fail          // 실패 (타이밍 미스 or 잘못된 입력)
├── Combo.Judge.Success       // 성공 (정상 입력)
└── Combo.Judge.GreatSuccess  // 대성공 (완벽 타이밍)

Food.Quality                  // 최종 음식 품질 (콤보 결과 합산으로 결정)
├── Food.Quality.Ruined       // 실패작 (Fail 비율 50% 이상)
├── Food.Quality.Normal       // 보통   (Success 과반)
├── Food.Quality.Good         // 수작   (GreatSuccess 과반)
└── Food.Quality.Masterpiece  // 걸작   (전부 GreatSuccess)

Customer                      // 손님 유형
├── Customer.Regular          // 일반 손님
├── Customer.Hungry           // 배고픈 손님 (타이머 빠름)
├── Customer.Picky            // 까다로운 손님 (대성공만 만족)
└── Customer.Critic           // 평론가 (실패 시 평판 -30)
```
