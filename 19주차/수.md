# UE5 비동기 에셋 로딩 & Data Asset

## 왜 필요한가?

하드 참조(일반 UObject 포인터)를 쓰면, 그 오브젝트가 로드될 때
참조된 에셋도 자동으로 전부 메모리에 올라온다.
에셋이 많아질수록 시작 시점에 메모리가 폭증하거나 히치(프리징)가 생긴다.

결론 : 참조와 실제 메모리 로드를 분리한다.

## 핵심 개념 비유

| 방식        | 비유                                                    |
| ----------- | ------------------------------------------------------- |
| 하드 참조   | 손님이 올 수도 있으니 가구를 전부 미리 집 안에 들여놓기 |
| 소프트 참조 | 가구점 주소만 적어두고 필요할 때 배송시키기             |
| 동기 로딩   | 배송 올 때까지 문 앞에서 게임이 멈춰 기다리기           |
| 비동기 로딩 | 게임은 계속 하면서 뒤에서 배송 오면 그때 꺼내 쓰기      |

## FSoftObjectPath vs TSoftObjectPtr

| 타입                | 역할                                        | 타입 제한         |
| ------------------- | ------------------------------------------- | ----------------- |
| `FSoftObjectPath`   | 에셋 경로(문자열)만 담는 날것의 구조체      | 없음              |
| `TSoftObjectPtr<T>` | FSoftObjectPath를 감싼 래퍼, 에디터 UI 제공 | T 타입으로 제한됨 |

## UDataAsset vs UPrimaryDataAsset

|                       | `UDataAsset`  | `UPrimaryDataAsset`         |
| --------------------- | ------------- | --------------------------- |
| Asset Manager 연동    | ❌            | ✅                          |
| 고유 ID               | 없음          | `FPrimaryAssetId` 있음      |
| 직접 로드/언로드 제어 | 어려움        | 가능                        |
| 사용 예               | 간단한 설정값 | 스킬, 아이템, 캐릭터 데이터 |

## 커스텀 AssetManager

**Epic 공식 권장 방식**

### UAssetManager

- 싱글톤, GEngine 아래에 존재
- FStreamableManager

### 언제 상속하나?

| 상황                               | 방법                      |
| ---------------------------------- | ------------------------- |
| 기본 PrimaryDataAsset 로딩만 필요  | 기본 UAssetManager 그대로 |
| 프로젝트 전용 로딩 로직, 캐싱 필요 | UAssetManager 상속        |
| 팀 프로젝트에서 로딩 API 통일      | 커스텀 클래스 권장        |

## 비동기 API

- 백그라운드 스레드 (ThreadPool) : 무거운 연산 수행, UObject 접근 절대 금지
- GameThread 마샬링 (AsyncTask(ENamedThreads::GameThread, ...)) : 완료/실패 콜백, UObject 접근 여기서만 허용

## 동기 vs 비동기 선택 기준

| 상황                        | 방식                | 이유                               |
| --------------------------- | ------------------- | ---------------------------------- |
| 로딩 화면, BeginPlay 초기화 | Synchronous         | 즉시 필요, 프레임 드랍 허용 구간   |
| 런타임 캐릭터/무기 스폰     | Asynchronous        | 프레임 블로킹 없이 백그라운드 로드 |
| UI 아이콘 프리로드          | Asynchronous        | 표시 전 미리 로드                  |
| 동기 로드 → 히치 발생       | Asynchronous로 전환 | Unreal Insights로 확인 후 교체     |

> ⚠️ 동기 로드는 렌더링 스레드까지 블로킹 → 게임플레이 중 호출하면 화면 멈춤(히치) 발생
