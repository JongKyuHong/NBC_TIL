# 팀프로젝트

## EDAssetManager

**Asset Bundle**

Primary Asset에 딸린 에셋들을 "이름 붙인 그룹"으로 묶는 기능
Primary Asset : 내가 로드 타이밍을 직접 제어하고 싶은 데이터 덩어리 (아이템 데이터, 캐릭터 설정, UWorld(맵))
Secondary Asset : 그 데이터가 참조하는 실제 리소스들 (메시, 사운드, 텍스처 등)

```c++
UPROPERTY(EditAnywhere, meta = (AssetBundles = "UI"))
TSoftObjectPtr<UTexture2D> Icon;

UPROPERTY(EditAnywhere, meta = (AssetBundles = "Gameplay"))
TSoftObjectPtr<USkeletalMesh> Mesh;
```

**LoadPrimaryAsset(AssetId, Bundles)**을 호출할때 동작 차이

번들이 빈 배열 -> Primary Asset 데이터 자체만 메모리에 올림, Icon이나 Mesh 파일은 로드 안됨
{"UI"} 지정 → Primary Asset + "UI" 번들에 속한 Icon 텍스처까지 함께 로드됨
{"UI", "Gameplay"} 지정 → Primary Asset + Icon + Mesh 모두 로드됨

**예시**

아이템 데이터를 예로들면

```
로비 화면  → LoadPrimaryAsset(ItemId, {"UI"})
            → 아이콘 이미지만 로드 (가볍게)

인게임 진입 → LoadPrimaryAsset(ItemId, {"UI", "Gameplay"})
            → 아이콘 + 실제 3D 메쉬까지 로드
```
