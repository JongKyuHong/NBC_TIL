# 멀티플레이 게임 개발

## RPC(Remote Procedure Call)

- 호출하는 PC와 실행하는 PC가 달라도 됨
- 게임에 영향이 적은 일시적인 효과들에 많이 사용
- 게임에 영향이 많다면 프로퍼티 레플리케이션

## 누군가 접속했을때 메시지 띄우기

- 플레이어가 접속할 떄마다 호출되는 함수는
  - 로그인 로직
- Server RPC, Client RPC, Multicast RPC중 어떤걸 호출?
  - 다른애들한테 다 알려줘야하므로 MulticastRPC를 호출
  - 게임모드의 로그인 로직에서 NewPlayer가 생기면 그의 GameState를 호출해서 (GetGameState로) MulticastRPC를 해주면 된다.
- Multicast RPC를 쓴다면 게임모드/스테이트 중 어디다가 써야할까?
  - 게임 스테이트
  - 왜냐하면 게임모드는 액터가 서버에만 있다. 서버에서만 실행, 게임모드에서 NetMulticast를 호출하면 서버에서만 실행된다.
  - 스테이트는 자동으로 기본값으로 레플리케이션 되기때문에 스테이트가 더 적합하다.
  - 모드/스테이트 둘다 베이스가 붙던 둘다 안붙던 해야된다.

## &ThisClass::

여기서 ThisClass는 언리얼매크로이다. 엔진이 자동으로 정의해주는 typedef로 현재 자기자신 가리킴
GENERATED_BODY() 매크로가 내부적으로 생성해줌
즉, GENERATED_BODY()가 있는 UObejct기반 클래스라면 모두 사용 가능

## 레플리케이션

생성된 액터의 정보를 네트워크 내 다른 클라이언트에게 복제하는 작업, 서버-클라이언트 구조에서는 서버에서 클라이언트로 복제되는것이 기본 규칙

두가지 방식이 있다.

- RPC
- Property Replication : 원하는 속성만 복제, 액터에 bReplicates = true를 지정해줘야함

# 코드카타

## 최대공약수

```c++
// https://www.acmicpc.net/problem/1850
#include <iostream>
#include <string>
#include <numeric>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(NULL);

    long long A, B;
    cin >> A >> B;

    long long answer = gcd(A, B);

    string tmp = string(answer, '1');

    cout << tmp;

    return 0;
}
```

- numeric아래에있는 gcd사용
- string에서 인자 위치 헷갈렸음
