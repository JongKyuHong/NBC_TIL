## 컴퓨터 구조의 전체적인 그림

- 메인보드 : 컴퓨터의 기반이 되는 회로판, CPU, 메모리, 시스템 버스등 주요 부품들이 장착
- 시스템 버스 : 데이터 이동 통로, 모든 구성요소를 연결, 데이터와 신호를 주고받게 한다.
- cpu : 컴퓨터의 두뇌, 산술논리연산장치,제어장치, 레지스터로 구성, 명령 해석하고 실행, 시스템버스로 다른 장치와 통신
- 메모리 : RAM, 시스텀 버스와 연결되어 CPU가 데이터를 빠르게 읽고 쓸 수 있다.
- 보조기억장치 : 전원이 꺼져도 데이터를 영구적으로 보관하는 저장 장치, 시스템 버스로 데이터가 이동
- 입출력장치 : 사용자와 컴퓨터 간의 상호작용을 담당, 모니터, 키보드, 마우스등이 포함, 시스템 버스를 통해 연결

## 데이터와 명령어

- 인코딩 : 사람이 읽는 문자를 컴퓨터가 이해하는 0, 1로 바꾸는 과정
- 디코딩 : 인코딩의 반대, 0과1을 사람이 읽을 수 있게

## 에셋 인코딩

- .png, .fbx는 엔진에 의해 최적화된 내부 포맷 .uasset으로 인코딩되어 저장

### 텍스처

- 색상을 숫자로 인코딩
- GPU가 시스템 버스를 통해 데이터를 빠르게 읽을 수 있도록 BC7, ASTC같은 특수 방식으로 인코딩하여 VRAM에 저장

### 메시

- 메시는 정점의 위치정보, 실수형 데이터로 인코딩

### 디코딩하기

- 하드웨어가 화면에 그림을 그리고 물리 연산을 하는 찰나의 순간에 디코딩이 일어남
- 메시 디코딩 : 인코딩된 0, 1의 부동 소수점 좌표값을 ALU(산술논리연산장치)가 읽어 들임
- 위치 계산 : 디코딩된 실수 데이터를 바탕으로 명령어에 따라 캐릭터의 위치를 계산하고 월드 좌표에 배치

### 컴파일언어, 인터프리터 언어

- 블루프린트는 인터프리터 방식으로 작동
- 엔진이 제공하는 블루프린트 네이티브화 기능은 인터프리터 노드들을 컴파일언어인 C++수준으로 변환해 성능을 올리는 기술

### 목적파일과 링킹

- 컴파일을해서 .obj파일을 만들었다고 해서 끝나는게 아님
- MyCharacter, MyGameMode가 있다고 했을때 얘네를 목적코드로 생성
- MyGameMode는 MyCharacter가 메모리 어디에 있는지 모름
- 그래서 이를 연결해주는 작업이 링킹
