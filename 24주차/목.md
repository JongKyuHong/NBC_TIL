## 최대값 최소값 탐색

max_element, min_element, minmax_element

```c++
int main()
{
    vector<int> v = {4, 1, 6, 3, 8, 2};

    // 1. max_element : 최대값의 위치를 찾아줌
    auto max_it = max_element(v.begin(), v.end());
    cout << "최대값 : " << *max_it << endl;

    // 2. min_element : 최솟값의 위치를 찾아줌
    auto min_it = min_element (v.begin(), v.end());
    cout << "최소값 : " << *min_it << endl;

    // 3. minmax_element : 최대 최소값을 한번에
    auto [min_iter, max_iter] = minmax_element(v.begin(), v.end());
    cout << "최대값2 : " << *max_iter << endl;
    cout << "최소값2 : " << *min_iter << endl;
}
```

## 특정 조건을 만족하는 원소 개수 구하기

count나 count_if를 사용하면 특정 조건을 만족하는 원소 개수를 구할 수 있음

```c++
int main()
{
    vector<int> v = {1, 1, 2, 2, 3, 3};

    int cnt = count(v.begin(), v.end(), 2);
    int even_cnt = count_if(v.begin(), v.end(), [](int x) {return x % 2 == 0;});

    cout << "값이 2인 원소 개수 : " << cnt << endl;
    cout << "짝수인 원소 개수 : " << even_cnt << endl;
}
```

count는 단순 값 비교만 가능하고 count_if는 사용자가 조건을 정의할 수 있음

## std::distance, std::advance

distance는 얼마나 멀리있나, advance는 그만큼 전진

```c++
vector<int> v = {1, 2, 3, 4, 5};
auto it = find(v.begin(), v.end(), 3);
cout << *it << endl; // 3
int idx = distance(v.begin(), it); // idx = 2;
advance(it, idx); // 2칸 전진
cout << *it << endl; // 5
```

## Q. N개의 정수가 주어지는데, 이 중 중복된 숫자는 제거하고, 오름차순으로 정렬하여 출력하시오. N은 최대 100,000이고, 정수는 최대 1,000,000까지 나온다고 합시다. (20분)

```c++
#include <iostream>
#include <string>
#include <vector>
#include <algorithm>
using namespace std;

int main()
{
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int N;
    cin >> N;
    vector<int> vec(N);
    for (int i = 0; i < N; ++i) cin >> vec[i];

    sort(vec.begin(), vec.end());

    auto it = unique(vec.begin(), vec.end());
    vec.erase(it, vec.end());

    for (int v : vec) cout << v << " ";
    cout << "\n";

    return 0;
}
```

## 멀티 스레드 환경이란?

하나의 프로세스 안에서 여러 실행 흐름(스레드)이 동시에 돌아가는 환경, 각 스레드는 독립적인 스택을 갖지만, 힙 메모리와 전역 변수는 모든 스레드가 공유

### 핵심 3대 문제

1. 경쟁 조건

   두 스레드가 동시에 같은 변수를 읽고 쓸 때 발생

```c++
int counter = 0;

// Thread A -> counter++ -> 내부적으로 읽기 -> 더하기 -> 쓰기 3단계
// Thread B -> 동시에 counter++ 실행
// 결과 쓰기라는 단계가 있기때문에 중간에 끼어들면 2가 되어야 하는데 결과적으로 1이 될 수 있음
```

2. 데드락

   스레드A가 락1을 점유하고 락2를 기다리고, 스레드B는 락2를 점유하고 락1을 기다리는 상황, 서로가 서로를 영원히 기다림

3. 메모리 가시성 문제(Visibility)

   컴파일러와 CPU는 성능을 위해 코드 실행 순러를 최적화(reordering) 함, 스레드 A가 변수를 수정해도 스레드B의 CPU캐시에는 반영이 안 될 수도 있음

### C++ 동기화도구

std::mutex -> <mutex>헤더 -> 용도 : 임계 영역 보호(락) -> 가장 기본
std::lock_guard -> <mutex> -> RAII방식 mutex관리 -> 예외 안전
std::unique_lock -> <mutex> -> 조건 변수와 함께 사용 -> 유연한 락
std::atomic<T> -> <atomic> -> 원자적 연산 -> 락 없이 안전
std::condition_variable -> <condition_variable> -> 스레드 대기/알림 -> 생산자-소비자 패턴

- Mutext란?

Mutual Exclusion(상호배제)의 약자, 한번에 딱 하나의 스레드만 공유 자원에 접근하도록 막아주는 자물쇠

```c++
#include <mutex>

std::mutex mtx;
int shared_data = 0;
void thread_func(){
    mtx.lock(); // 자물쇠 잠금
    shared_data++;  // 공유자원 접근
    mtx.unlock();   // 자물쇠 해제
}

// 위처럼 구성하면 lock하고나서 예외가 터지면 unlock이 영원히 안불림 그래서 lock_guard를 쓴다.

void thread_func2(){
    std::lock_gurad<std::mutext> lock(mtx); // 생성되는 순간 자동으로 lock()
    shared_data++; // 함수가 끝나거나 예외가 터져도 lock이 자동 해제됨(RAII 패턴)
}
```

- 임계구역 : 여러 스레드가 동시에 실행하면 절대 안되는 코드 영역, Mutex로 이 구간을 보호

```c++
// 임계 구역 예시 — counter++는 원자적 연산이 아님
int counter = 0;

void bad_func() {
    counter++;  // ← 여기가 임계 구역
    // 내부적으로: 읽기 → 더하기 → 쓰기 (3단계)
    // 두 스레드가 동시에 실행하면 데이터 손실 발생
}

void good_func() {
    std::lock_guard<std::mutex> lock(mtx); // 자물쇠
    counter++;  // ← 이제 한 번에 하나의 스레드만 실행
}
```

### 멀티스레드에서 안전한 자료구조

1. std::atomic<T> : 단순 타입의 원자적 변수

   락을 쓰지않고CPU레벨에서 원자성을 보장하는 가장 빠른 방법

   ```c++
   #include <atomic>

   std::atomic<int> counter = 0;

   // 어떤 스레드에서 호출해도 안전
   counter++;  // 원자적 증가
   counter.fetch_add(5);   // 원자적 덧셈
   int val = counter.load(std::memory_order_acquire); // 안전한 읽기
   counter.store(10, std::memory_order_release); // 안전한 쓰기
   ```

   std::atomic은 int, bool, 포인터 같은 단순 타입에만 적용됨, std::vector같은 복잡한 타입에는 쓸 수 없음

2. Mutex로 감싼 Thread-Safe Queue(생산자 소비자 패턴)
   게임에서 작업큐(Task Queue)나 메시지 큐 구현 시 가장 많이 쓰이는 패턴

   ```c++
   #include <queue>
   #include <mutex>
   #include <condition_varibale>
   #include <optional>

   template<typename T>
   class ThreadSafeQueue{
    public:
        // 데이터 넣기 (생산자가 호출)
        void push(T value){
            std::lock_guard<std::mutex> lock(mutex_);
            // 잠금 - 다른 스레드가 동시에 push/pop 못하게
            queue_.push(std::move(value));
            cv_.notify_one(); // 대기중인 소비자 스레드 하나 깨우기
        }

        // 데이터 꺼내기 - 없으면 생길 때까지 대기 (소비자 호출)
        T wait_and_pop(){
            std::unique_lock<std::mutex> lock(mutex_);
            cv_.wait(lock, [this](return !queue_.empty();));
            // queue가 비어있으면 여기서 대기, push가 notify하면 깨어남

            T val = std::move(queue_.front());
            queue_.pop();
            return val;
        }

        // 데이터 꺼내기 - 없으면 그냥 빈 값 반환 (블로킹 없음)
        std::optional<T> try_pop(){
            std::lock_guard<std::mutex> lock(mutex_);
            if (queue_.empty()) return std::nullopt;
            T val = std::move(queue_.front());
            queue_.pop();
            return val;
        }
    private:
        std::queue<T> queue_;
        std::mutex mutex_;
        std::condition_variable cv_;
   }

   ```

3. Thread-Safe Stack

   ```c++
   #include <stack>
   #include <mutex>
   #include <optional>

   template<typename T>
   class ThreadSafeStack {
   public:
       // 데이터 넣기
       void push(T value){
           std::lock_guard<std::mutex> lock(mutex_);
           stack_.push(std::move(value));
       }

       // 데이터 꺼내기 - 없으면 빈 값 반환
       std::optional<T> pop(){
            std::lock_guard<std::mutex> locK(mutex_);
            if (stack_.empty()) return std::nullopt;
            T val = std::move(stack_.pop());
            stack_.pop();
            return val;
       }

       bool empty() const {
            std::lock_guard<std::mutex> locK(mutex_);
            // mutable이어야 const 함수에서 lock가능
            return stack_.empty();
       }
   private:
       std::stack<T> stack_;
       mutable std::mutex mutex_;
   }
   ```

## 멀티스레드 환경에서 안정적으로 사용할 수 있는 자료구조를 설명해주세요

멀티스레드 환경에서는 여러 스레드가 힙 메모리와 전역변수를 공유하기 때문에, 동시게 같은 자원에 접근하면 경쟁 조건(Race Condition)이 발생함. 이를 해결하는 원리가 Mutex(상호배제)임 한번에 하나의 스레드만 공유자원에 접근할 수 있도록 자물쇠 역할을 하며, C++에서는 std::mutex와 lock_guard를 함께써서 RAII방식으로 안전하게 락을 관리
이를 통해서 Thread-Safe Queue와 Stack을 만들 수 있음, Queue는 mutex로 push/pop 임계구역을 보호하고, condition_variable로 데이터가 없을때 소비자 스레드를 대기 시킴, stack은 동일하게 mutex로 감싸고, pop시 빈 경우를 std::optional로 처리
단순 카운터나 플래그처럼 단일값을 공유할때는 mutex없이 std::atomic<T>를 사용하면 더 가볍고 빠름
