# 면접 질문

## 리플렉션에 대해 설명

프로그램이 런타임에 자기 자신을 조사하고 조작할 수 있는 기능, "프로퍼티 시스템"이라고도 불림

### 왜 필요한가?

일반 C++은 런타임 정보(RTTI)를 거의 제공하지 않음, 그래서 언리얼은 별도의 매크로 기반 시스템을 구축해 클래스, 함수, 변수의 메타데이터를 수집하고 엔진이 이를 런타임에 활용할 수 있도록 함. 이 덕분에 에디터의 디테일 패널, 블루프린트 연동, 가비지 컬렉션, 네트워크 리플리케이션, 직렬화 등이 모두 작동

### 동작 원리 UHT(Unreal Header Tool)

리플렉션 시스템은 선택적(opt-in)임, 즉 엔진에게 알리고 싶은 클래스/함수/변수에만 매크로를 붙여야 함. 빌드 시 UHT가 헤더파일에 달린 매크로들을 읽어 메타데이터를 수집하고, `FileName.generater.h` 파일을 자동 생성, 따라서 헤더 파일 최하단 include는 반드시 #include "FileName.generated.h" 여야함

### 핵심 매크로 5가지

UCLASS() : 클래스, UObject 파생 클래스를 엔진에 등록, 블루프린트 부모 클래스로 노출 가능
USTRUCT() : 구조체, 구조체 메타데이터 등록, 블루프린트에서 사용 가능
UENUM() : 열거형, 열거형을 엔진에 노출, 블루프린트 드롭다운 등으로 활용
UPROPERTY() : 멤버 변수, 변수를 에디터 노출, 직렬화, GC, 네트워크 복제 대상으로 등록
UFUNCTION() : 멤버 함수, 함수를 블루프린트 호출, RPC, 이벤트 대상으로 등록

### 리플렉션이 활성화하는 기능

- 가비지 컬렉션 : UPROPERTY로 등록된 변수만 GC가 참조 여부를 추적함, 등록하지 않으면 엔진이 객체를 임의로 파괴할 수 있음
- 블루프린트 연동 : UCLASS, UFUNCTION, UPROPERTY에 BlueprintCallable, BlueprintReadWrite등의 지정자를 붙여야 블루프린트에서 접근할 수 있음
- 에디터 디테일 패널 : EditAnywhere, EditDefaultsOnly 등의 지정자를 통해 에디터 UI에서 변수값을 직접 편집할 수 있음
- 네트워크 리플리케이션 : UPROPERTY(Replicated) 또는 UFUNCTION(Server, Reliable) 형태로 서버-클라이언트 동기화 대상을 지정함
- 직렬화(Serialization) : 리플렉션 정보를 바탕으로 오브젝트 상태를 저장, 불러오기 함

### UClass 내부구조

리플렉션이 등록된 클래스는 엔진이 UClass 객체를 자동으로 생성해 관리함, 이 UClass 안에는 CDO(Class Default Object)라는 기본값 사본이 하나 있으며, 새 인스턴스를 생성할 때 이 CDO를 복사해 초기화한다. 런타임에 GetClass()나 StaticClass()를 호출하면 이 UClass에 접근할 수 있음
