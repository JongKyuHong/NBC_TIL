# 면접 개념

## 프로세스 메모리 구조를 설명해주세요

### Code 영역

실행할 프로그램의 소스코드가 저장되는 영역, 읽기 전용으로 지정되어 있어서 코드를 실행중에 수정할 수 없고, 크기는 컴파일 시점에 고정
함수, 조건문, 반복문이 모두 이 영역에 올라감

### Data 영역

초기값이 있는 전역변수와 static 변수가 저장됨, 프로그램 시작 시 할당되고 종료 시 해제되며, 크기는 컴파일 시점에 결정

```c++
int total = 0; // Data 영역 (초기값 있음)
```

### BSS영역

초기값이 없는 전역 변수와 static 변수가 저장되며 자동으로 0으로 초기화됨, Data영역과 논리적으로 분리되어 있지만 같은 Data세그먼트로 묶어서 설명하기도 함

```c++
int count; // BSS영역(초기화 없음, 자동으로 0)
```

### Heap영역

런타임에 동적으로 할당하는 메모리 영역, C++에서는 new 키워드로 할당하고 delete로 해제하며, 프로그래머가 직접 관리해야 함, 낮은 주소 -> 높은 주소 방향으로 메모리가 증가함

```c++
int* arr = new int[10]; // Heap에 할당
delete[] arr; // 직접 해제 필수
```

### Stack영역

함수 호출 시 생성되는 지역변수, 매개변수, 반환 주소가 저장되는 영역. 함수가 종료되면 자동으로 해제되며 LIFO 방식으로 관리됨, Heap과 반대로 높은 주소 -> 낮은 주소 방향으로 증가함

```c++
void dfs(int depth, string tmp){
    int gap = 0; // Stack에 할당, 함수 종료 시 자동 해제
}
```

## Unreal GC 동작 원리

언리얼 GC는 `UObject` 기반 객체만 관리하며, Mark and Sweep 알고리즘을 사용함
C++ 기본언어는 GC를 지원하지 않지만, 언리얼이 자체적으로 리플렉션 시스템을 기반으로 GC를 구현했음

Mark and Sweep 흐름

1. 모든 UObject를 `Unreachable`를 상태로 초기화 - 일단 전부 죽은것으로 간주
2. Root Set부터 참조 탐색 - `GEngine, GWorld` 같은 루트 객체부터 시작해 `UPROPERTY`로 선언된 포인터를 따라가며 도달 가능한 객체의 `Unreachable` 플래그 제거
3. Sweep - 탐색후에도 `Unreachable`상태로 남아있는 객체를 메모리에서 해제
4. 멀티스레드 병렬 처리 - 언리얼 GC는 멀티스레드 기반으로 병렬 수행하여 성능을 최적화

기본 설정기준 60초마다 한번 GC사이클이 실행된다. 설정으로 변경할 수 있음

### UPROPERTY가 핵심인 이유

GC는 UPROPERTY() 매크로로 선언된 포인터만 추적대상으로 인식, 매크로 없이 raw포인터로 들고 있으면 GC가 해당 참조를 모르기 때문에, 참조중인 객체가 GC에 의해 삭제되어 댕글링 포인터가 발생할 수 있음

참조는 하되 GC를 막고 싶지 않으면 TWeakObjectPtr를 사용, 대상 객체가 GC로 파괴되면 자동으로 nullptr처럼 동작하므로 댕글링 포인터 없이 안전하게 약한 참조를 유지할 수 있음

# RAM의 특징

1. 휘발성 저장 장치

RAM의 가장 큰 특징은 전원이 차단되면 저장된 명령어와 데이터가 모두 사라진다는 점

2. 보관과 실행의 분리

CPU는 보조기억장치에 직접 접근하여 프로그램을 실행할 수 없습니다.
보조기억장치 : 보관할 대상을 영구적으로 저장하는 거대한 창고 역할
RAM : 보조기억장치에서 실행할 대상을 복사해와 CPU가 즉각 처리할 수 있도록 돕는 작업대

# 운영체제

운영체제는 자원관리자, 커널영역에 산다. 정부역할(메모리 관리, CPU관리, 입출력 장치 관리)
개발자는 운영체제를 알아야 한다. 운영체제가 없다면 하드웨어를 조작하는 코드를 개발자가 직접 다 작성해야 하므로

## 커널

운영체제의 핵심적인 기능을 담당하는 부분

프로세스 관리 : 실행 중인 프로그램들을 조율
자원 할당 : CPU, 메모리, 하드디스크 같은 부품들을 어떤 프로그램에 얼마나 나눠줄지 결정
파일 시스템 관리 : 우리가 만드는 파일과 폴더를 안전하게 저장하고 찾아줌

사용자가 커널에 명령을 내릴 수 있도록 도와주는 통로가 UI, GUI, CLI등이 있음

## 이중 모드와 시스템 호출 (Dual Mode & System Calls)

운영체제는 실행하는 응용 프로그램이 하드웨어 자원에 직접 접근하는것을 철저히 방지
만약 게임이나 메모장 같은 프로그램이 CPU나 하드디스크를 마음대로 조작할 수 있다면 작은 실수 하나에도 컴퓨터 전체가 고장나거나 중요한 데이터가 사라지는 무질서한 상태가 될 수 있음

1. 운영체제는 든든한 문지기

운영체제는 응용 프로그램이 자원에 접근하려고 할 때 반드시 자신을 거쳐 가도록 관리하는 문지기 역할을 함

### CPU의 실행 상태 : 이중모드

이러한 문지기 역할은 CPU가 명령어를 실행하는 모드를 두 가지로 나누는 이중 모드로 구현

1. 사용자 모드
   - 일반적인 응용 프로그램이 실행되는 기본 모드
   - 하드웨어 자원에 직접 접근하는 명령어를 실행할 수 없음
   - 이 모드에서는 운영체제의 서비스를 직접 제공받을 수 없음

2. 커널 모드
   - 운영체제가 실행되는 모드, 모든 명령어를 실행할 수 있는 특권 모드
   - 자원에 접근하는 명령어를 포함해 하드웨어를 완전히 제어할 수 있음

### 시스템 호출

사용자 모드에서 작동중인 프로그램이 하드디스크에 데이터를 저장하는 등의 작업을 하려면 시스템 호출이 필요함

- 정의 : 응용 프로그램이 운영체제에 커널 서비스를 요청하는 특별한 통로
- 작동 원리 : 프로그램이 시스템 호출을 발생시키면 CPU는 실행중인 작업을 백업하고 커널 모드로 전환, 운영체제는 내부의 코드를 실행하여 요청받은 작업을 대신 처리한 뒤, 다시 사용자 모드로 돌아와 프로그램을 실행
- 특징 : 시스템 호출은 일종의 소프트웨어 인터럽트와 같아서, 하드웨어 인터럽트와 유사한 순서로 처리

```
우리가 은행(하드웨어 자원)에 갔다고 생각해보세요. 금고 안에 있는 돈을 마음대로 꺼낼 수는 없죠?
우리는 창구 직원(운영체제)에게 요청서(시스템 호출)를 작성해서 제출해야 합니다.
직원은 우리의 신분을 확인하고 안전한지 판단한 뒤, 직접 금고에 들어가 작업을 처리하고 결과를 알려줍니다.
```

## 운영체제 핵심 서비스

운영체제의 핵심인 커널이 제공하는 대표적인 서비스는 프로세스 관리, 자원 접근 및 할당, 파일 시스템 관리 세가지

### 프로세스 관리

실행 중인 프로그램을 프로세스라고 부름, 윈도우의 작업관리자를 보면 수 많은 프로세스가 동시에 실행되고 있는것을 볼 수 있음

- 멀티 태스킹의 원리 : 일반적으로 하나의 CPU는 한 번에 하나의 프로세스만 실행할 수 있음, 운영체제는 여러 프로세스를 아주 짧은 시간동안 번갈아 가며 실행하여 마치 동시에 작동하는 것 처럼 보이게 함
- 다양한 상태 관리 : 어떤 프로세스는 입출력 장치를 주로 쓰고, 어떤 프로세스는 CPU만 집중적으로 사용
- 고급 관리 기술: 여러 프로세스가 동시에 실행될 때 데이터가 꼬이지 않게 하는 프로세스 동기화와, 프로세스들이 서로 자원을 기다리며 멈춰버리는 교착 상태(Deadlock) 해결도 운영체제의 중요한 임무

### 자원 접근 및 할당 (Resource Allocation) 🏗️

모든 프로세스는 실행을 위해 컴퓨터 부품인 자원이 필요하며, 운영체제는 이 자원들을 공정하고 효율적으로 나눠줌

- CPU 스케줄링: 어떤 프로세스에게 CPU를 먼저 줄지, 얼마나 오래 쓰게 할지를 결정
- 메모리 관리: 프로세스들이 메모리의 어느 주소에 적재될지 결정합니다. 메모리가 꽉 차서 공간이 부족한 상황이 생겨도 이를 극복하고 프로그램을 실행할 수 있게 관리
- 입출력 장치 관리: 키보드나 마우스 같은 장치가 보내는 신호(인터럽트)를 처리하기 위해 커널 영역에 있는 인터럽트 서비스 루틴을 제공하여 입출력 작업을 수행

### 파일 시스템 관리

우리가 컴퓨터를 쓰면서 파일을 열고, 만들고, 삭제하는 모든 행동은 운영체제의 서비스 덕분입니다.

- 정리 정돈: 관련 있는 파일들을 묶어 디렉터리(폴더)로 관리할 수 있게 해줌
- 보조기억장치 관리: 하드디스크나 SSD 같은 보조기억장치 속의 데이터를 우리가 인식하기 쉬운 파일 형태로 관리하는 것도 운영체제의 핵심 기능

# 핵심 요약

1. 운영체제의 핵심 서비스를 제공하는 부분은 커널
2. 응용 프로그램이 커널의 서비스를 받으려면 시스템 호출을 통해 커널 모드로 전환해야함
3. 주요 서비스에는 프로세스 관리, 자원 할당, 파일 시스템 관리가 있음
