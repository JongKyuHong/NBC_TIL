# 모듈

- 언리얼 소프트웨어 아키텍처의 가장 기본적인 구성 요소, 특정 에디터 툴, 런타임 기능, 라이브러리 등의 기능을 독립된 코드 단위로 캡슐화

## 모듈 사용의 이점

- 함수 기능을 캡슐화하고 코드 내부를 숨김
- 모듈은 각기 다른 컴파일 유닛으로 컴파일되기때문에 변경된 모듈만 컴파일하므로 대규모 프로젝트의 빌드 시간이 대폭 단축
- 컴파일하는 동안 프로젝트에 사용하지 않는 모듈을 안전하게 제외
- 런타임 시 지정한 모듈의 로드 및 언로드 시간을 조절할 수 있다.
- 사용할 수 있거나 활성화할 시스템을 관리하여 프로젝트의 퍼포먼스를 최적화 할 수 있다.

## 모듈 구성하기

1. 프로젝트의 Source 폴더 최상위 경로에 모듈용 디렉터리를 생성, 모듈과 이름이 같아야 함
2. 모듈의 루트 디렉ㅌ리에 `[ModuleName].Build.cs` 파일을 생성하고, 이 파일로 다른 모듈과의 종속성을 정의, 종속성을 정의하면 언리얼 빌드 시스템에서 모듈을 발견할 수 있다.
3. 모듈 루트 디렉터리에 `Private`과 `Public` 서브 폴더를 생성
4. 모듈의 `Private` 서브 폴더에 `[ModuleName]Module.cpp`파일을 생성, 이 파일로 모듈 및 언리얼 엔진에서 모듈 관리를 위해 사용하는 일반 함수를 시작 및 중단 가능
5. 모듈을 언제 어떻게 로드할지 정하기 위해 `.uproject` 또는 `.uplugin` 파일에 속한 모듈에 환경설정 정보를 추가
6. 이 모듈을 사용해야 하는 모든 모듈의 `Build.cs` 파일에 종속성으로 모듈을 나열한다.
7. `[ModuleName].Build.cs` 파일을 변경하거나 소스 파일을 다른 폴더로 옮길 때마다 해당 IDE의 솔루션 파일을 생성 (우리 늘하던 uproject에다가 프로젝트 파일생성 하는거)

## Build.cs파일에서 종속성 설정하기

- 언리얼 빌드 시스템은 IDE에 대한 솔루션 대신 프로젝트의 `Target.cs` 파일과 모듈의 `Build.cs` 파일에 따라 프로젝트를 빌드
- IDE 솔루션은 코드를 편집할때 자동생성되지만, 언리얼 빌드 툴(UHT)은 프로젝트를 컴파일할 때 IDE 솔루션을 무시
- 언리얼 빌드 시스템에서 솔루션을 인식하도록 하려면 모든 모듈의 루트 디렉터리에 [ModuleName].Build.cs파일을 배치해야함
- 모듈을 `ModuleRules` 클래스에서 상속받은 클래스로 정의해야함

## 프라이빗 및 퍼블릭 종속성

- 퍼블릭 .h 파일처럼 모듈에 있는 클래스를 퍼블릭으로 사용할 경우 `PublicDepedencyModuleNames` 목록을 사용해야 함
- 이 모듈에 의존하는 다른 모듈도 헤더 파일을 문제없이 추가할 수 있다.

- `.cpp`파일처럼 모듈을 프라이빗으로만 사용할 경우 `PrivateDepedencyModuleNames` 목록에 추가해야 함
- 가능하면 프로젝트의 컴파일 시간을 줄여주는 프라이빗 종속성을 쓰는게 좋다.
- 헤더파일 내에서 전방 선언을 사용하여 수많은 종속성을 퍼블릭이 아닌 프라이빗으로 만들 수 있다.

## 종속성

- A모듈이 B모듈의 코드를 사용하려면 B에 종속된다고 한다.

```c++
// A모듈의 Build.cs

PublicDependencyModuleNames.AddRange(new string[] { "B" });
PrivateDependencyModuelNames.AddRange(new string[] { "C" });
```

```c++
// D모듈의 Build.cs
PublicDependencyModuleNames.AddRange(new string[] { "A" });
```

- A모듈에 종속된 D가 있을때
- Public에 있는 B는 D에서도 사용가능
- Private에 있는 C는 D에서 접근 불가

## C++에서 모듈 구현하기

- `IModuleInterface`를 확장하는 클래스를 생성하고 `IMPLEMENT_MODULE` 매크로에 추가

```c++
#include "Modules/ModuleManager.h"

IMPLEMENT_MODULE(FDefaultModuleImpl, ModuleTest);

// h
class FMyModule : public IModuleInterface
{
public:
    virtual void StartupModule() override;   // 모듈 로드될 때 자동 호출
    virtual void ShutdownModule() override;  // 모듈 언로드될 때 자동 호출
};
```

- `FDefaultModuleImpl` 은 `IModuleInterface` 를 확장하는 빈 클래스
- 모듈 로드/언로드시에 할게있으면 직접 클래스 만들어서 `FDefaultModuleImpl` 자리에 넣으면 됨

## 프로젝트에서 모듈 사용하기

- 언리얼 빌드 툴은 컴파일 속도를 최적화하기 위해 프로젝트의 종속성 체인에서 발견한 모듈만 컴파일 함

# 플러그인

언리얼 엔진에 특정 함수기능을 추가해주는 선택적 소프트웨어 컴포넌트, 플러그인을 이용해 언리얼 코드를 직접 수정하지 않고 완전히 새로운 기능을 추가하거나 내장된 기능을 수정할 수 있다.
