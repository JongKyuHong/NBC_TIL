# CPU아키텍처와 게임 루프

## CPU의 내부 구조와 핵심 부품

CPU는 메모리에 저장된 명령어를 읽어들이고(Fetch), 해석하고(Decode), 실행하는(Execute)장치
이 과정은 세가지 핵심 부품인 연산, 제어, 기억장치의 상호작용으로 이루어진다.

### ALU (Arithmetic Logic Unit)

CPU내에서 실제로 값이 변하는 장소
기술적 정의 : 가산기, 보수기, 논리 게이트 등으로 구성된 회로 뭉치
수행 업무 : 레지스터가 들고 있는 숫자들을 가져와서 더하거나, 빼거나, 크기를 비교, 계산 결과를 레지스터에 기록
CPU의 실제 계산기, CPU의 나머지 부분은 전부 ALU를 잘 돌리기 위해 존재, 데이터가 아무리 돌아다녀도 실제로 값이 바뀌는 곳은 여기뿐, 레지스터 정확히는 플래그 레지스터에 기록하는 것은 계산이 끝나자마자 방금 계산 결과가 마이너스였다 하고 알림판을 들어서 알려주는 것

### 제어장치 (CU, Control Unit)

전기 신호를 뿌려 CPU의 모든 부품을 깨우고 움직이게 함
클럭 : 박자 맞추기
명령어 해독기 : 메모리에서 가져온 숫자(기계어)가 더하기인지 저장인지 번역
제어 버스 : 번역된 결과에 따라 ALU에게 전기 신호를 보냄

### 레지스터 (Registers)

메모리까지 가기엔 너무 멀어서 CPu가 자기 품에 들고있는 데이터 바구니

레지스터 명칭
PC(Program Counter) : 다음 요리 대기표, 요리사가 한가지 요리를 끝내고 다음엔 뭐하지? 하고 헤매지 않게 다음에 할일의 번호를 미리 적어두는 포스트잇

IR(Instruction Register) : 지금 보고있는 레시피, 요리사가 지금 당장 실행 중인 요리법 한 문장을 담아두는 곳, 이 글자를 읽으면서 손을 움직임

MAR (Memory Address Register) : 창고 주소 쪽지, 거대 창고(메모리)에 재료를 가지러 갈때 주소만 적어서 들고있는 쪽지

MBR (Memory Buffer Register) : 임시 배차 쟁반, 창고에서 꺼내온 재료가 주방(CPU)안으로 들어오기 위해 잠시 거쳐 가는 쟁반, 모든 재료는 이 쟁반을 통해서만 들어오고 나감

범용 레지스터 : 손바닥 위의 그릇, 소금 한 꼬집, 설탕 한 스푼처럼 지금 당장 계산에 쓸 숫자를 담아두는 작은 그릇, 요리사가 가장 자주 손을 뻗는 곳

### 데이터 흐름 시나리오

1. PC가 '10번지로가!' 주소를 알려줌 (준비)
2. MAR/MBR을 통해 메모리 10번지의 명령어를 가져와서 IR에 넣음 (인출)
3. 제어장치가 IR을 보고 아 더하기를 하는구나 해석 (해독)
4. 제어장치가 ALU에 전기를 보내고, ALU는 범용 레지스터의 숫자를 더함 (실행)
5. 계산 결과가 0이면 플래그가 번쩍 들리고, 다음 일을 위해 PC 숫자가 올라감 (반복)

## 시스템 버스

부품을 잇는 정보의 통로
CPU, 메모리, 주변 장치들이 전기 신호를 주고받는 하드웨어적 통로

### 주소 버스

CPU가 `어디에 있는 데이터`를 쓸 것인지 지목하는 통로
기능 : CPU가 읽거나 쓰려는 메모리의 주소 정보를 전달
특징 : 보통 CPU에서 메모리 방향으로만 흐르는 단방향 통로
성능의 척도 : 주소 버스의 폭(선로 수)이 CPU가 한 번에 지정할 수 있는 메모리 용량을 결정

마치 편지쓸때 주소적는것과 비슷, 메모리라는 거대한 창고에서 특정 데이터를 찾기위해

### 데이터 버스

실제 내용물이 이동하는 통로

기능 : CPU와 메모리, 혹은 주변장치 간에 실제 명령어와 데이터를 주고받음
특징 : 양방향으로 데이터가 이동해야하므로 양방향 통로
성능의 척도 : 데이터 버스의 폭이 한 번에 전송할 수 있는 데이터의 양(32비트, 64비트)을 결정

### 제어 버스

상대방에게 무엇을 할지 명령하고 상태를 확인하는 신호등
기능 : 읽어라, 써라 같은 제어 명령이나 장치의 상태를 알리는 신호를 전달
주요 신호 :
메모리 읽기/쓰기 : 메모리에 데이터를 넣을지 뺄지 결정
인터럽트 : 주변 장치가 CPU에 긴급한 용무가 있음을 알림
버스 승인 : 버스를 사용해도 좋다는 허락을 보냄

주소와 데이터가 준비되어도 언제 무엇을 할지 정해주지 않으면 아무 일도 일어나지 않음, 제어 버스는 지금 당장 읽어라 혹은 지금 적힌 데이터를 저장해 라고 신호를 주는 지휘봉, 부품들이 일을 마쳤는지 오류가 났는지 보고받는 무전기 역할도 겸

### 버스가 협력하는 실제 과정

CPU가 메모리 100번지의 데이터를 읽고 싶을 때, 세 버스는 이렇게 움직임

- 주소 버스 : 100번지의 데이터로 가라 (위치 지정)
- 제어 버스 : 데이터를 읽어 (명령 하달)
- 데이터 버스 : (명령을 받은 메모리가) 100번지 데이터를 CPU로 보냄 (내용물 이동)
