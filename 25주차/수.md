1. Struct와 Class의 차이점

기본 접근 제어자가 Struct는 publie, Class는 private이다.

- 상속시에도 동일하게, 현대 C++에서는 관습적으로 Struct를 순수한 데이터만 담는 경우에 사용하고, Class는 객체 지향의 다형성이나 복잡한 로직 구현에 사용

2. 포인터와 참조의 차이점을 설명하고 각각 언제 사용하는것이 좋을까?

포인터는 메모리주소를 가리키는 변수 추가적인 메모리를 사용함, 참조는 변수에 대한 별칭 선언시 초기화를 무조건 해주어야함

- 포인터는 nullptr을 가질 수 있고, 가리키는 대상을 언제든지 바꿀수있다(재할당 가능), 참조는 대상을 바꿀 수 없다. 가리키는 대상이 없을 수 있거나 대상을 바꿔야 한다면 포인터, 절대 null일수없고 안전하게 별칭으로 쓰고싶으면 참조

3. lvalue와 rvalue의 개념

lvalue는 말그대로 왼쪽 값, 값이 들어가는 변수명을 뜻함, rvalue는 오른쪽값 어떤 변수에 들어가는 값을 뜻함

- lvalue는 메모리 주소를 가지고 있어 이름으로 접근 가능한 지속적인 객체, rvalue는 표현식이 끝나면 사라지는 임시적인 값

4. C++에서 변수와 상수의 차이점

변수는 변할수있는값이고 상수는 고정된값

- const는 런타임에 결정되는 상수를 포함, constexpr은 컴파일타임에 값이 결정되어 최적화에 유리하다.

5. 스택과 힙 메모리 영역에 대한 차이를 설명

스택의 경우는 함수의 매개변수, 함수내에서 선언된 지역변수 등등이 저장되는 공간이고 알아서 해제가 된다. 힙 메모리의 경우에는 동적으로 생성한 값들이 들어가는 공간이고 본인이 직접 해제해주어야 한다.

6. 생성자와 소멸자의 역할은 무엇인가

생성자는 인스턴스가 생성될때 값을 초기화해주는 역할을 한다. 소멸자는 인스턴스가 삭제될때 메모리공간을 비워주는 역할을 한다.

- 소멸자는 자원해제를 한다.

7. inline함수는 언제 사용하며 컴파일러가 실제로 어떻게 처리하는가?

inline함수는 컴파일러에게 함수 호출 대신 함수의 본체코드를 호출 지점에 직접 삽입해달라고 요청하는 함수

- 컴파일 타임에 함수 호출 오버헤드(스택 프레임 생성, 복귀 주소 저장등)을 없애기 위해 코드를 복사해서 붙여넣는 식으로 처리, 컴파일러가 거부할 수도있음

8. C++에서 함수 포인터를 사용하는 예제를 작성하고 이유를 설명

```c++
int Add(int a, int b){
    return a + b;
}
int Mux(int a, int b){
    return a*b;
}
int (*funPtr)(int, int);
funPtr = Add;
cout << funPtr(10, 20) << endl; // 30;
funPtr = Mux;
cout << funPtr(10, 20) << endl; // 200;
```

전략패턴처럼 런타임에 실행 함수를 교체해야 하거나, 콜백 함수를 전달할때 필수적

9. C++에서 함수 오버로딩은 무엇인가?

이름이 같고 인자의 개수나 종류가 다른 함수를 재정의하는것을 오버로딩이라고 합니다.

10. C++에서 std::array와 C배열의 차이점 2가지

- std::array는 자신의 크기를 스스로 알 고 있음(size()), C배열은 포인터로 쉽게 퇴화(Decay)되어 크기 정보를 잃어버리기 쉬움

11. C++의 RAII의 개념을 설명하고 장점을 설명해보세요.

- 객체가 생성될때 자원이 할당되고, 객체가 소멸될때 자원을 자동으로 해제하는 기법, 예외가 발생하여 함수가 갑자기 종료되어도, 스택에 쌓인 객체의 소멸자가 호출되므로 메모리 누수를 방지할 수 있다. (스마트 포인터)

12. 실수 자료형 사용시 발생 문제 (부동 소수점 오차 등)를 설명해주세요

컴퓨터는 이진수를 사용하기 때문에 실수 자료형의 소수점 아래 수를 완벽하게 표현할 수 없다. 부동소수점이 그 예시인데 이진수로 표현하게되면 조금씩의 오차가 생김

## STL

1. vector에서 push_back을 하면 어떤일이 벌어질까?

vector의 맨뒤자리에 새로운 값이 추가된다.

2. 전위 ++it와 후위 it++의 동작차이를 설명

전위의 경우 값을 증가시키고 리턴시키는 동작이 연산자 내부적으로 되어있는데 후위의 경우 이전값을 저장해두고 값을 증가 시킨 뒤 이전값을 반환해주는 식이기 때문에 int같은 값들은 성능차이가 크게없으나 이터레이터같은경우는 성능차이가 생길 수 있어 전위증감 연산자를 사용하는것이 낫다.

3. unique_ptr과 shared_ptr의 차이

unique_ptr은 소유권을 독립적으로 가지고 있는것, 소유권 이전만 가능, shared_ptr은 소유권을 여러군데에서 가질 수 있음, 순환참조를 조심해야 한다.

- shared_ptr의 경우 내부적으로 레퍼런스 카운트를 관리하는 Control Block을 공유한다.

4. vector의 size와 capacity의 차이

size는 실제로 값이 들어있는 개수고 capacity는 vector의 수용값, vector의 수용값에 size가 거의 다다르면 resize가 필요함 근데 이게 성능을 좀 잡아먹는다. 메모리 새로운공간도 찾고 복사도 해야하므로

5. find()와 binary_search()의 차이

find의 경우 정렬되지 않은 데이터에서도 사용할 수 있음, binary_search는 정렬된 데이터에서만 사용할 수 있다.

- find는 선형탐색에 반복자를 반환함, binary_search는 bool값을 반환

6. pair와 tuple의 차이점

pair는 딱 두가지 값만 가지는 경우 사용할 수 있음 .first와 .second로 접근. tuple은 여러가지 값을 가지고 있는 경우에도 사용할 수 있다.

7. STL list::sort가 따로 존재하는 이유

연결리스트의 경우 sort하는 방식이 일반적인 컨테이너들과는 달라서 그렇다. \

- std::sort는 임의 접근 반복자(Random Access Iterator)를 요구하는데 list는 양방향 연결 리스트라 이를 지원하지 않는다.

8. std::map에 대해 설명

map의 경우 내부적으로 레드블랙트리로 되어있다. 따라서 자기가 균형을 잡을 수 있음 -> map은 키값을 기준으로 정렬이 된다. 데이터는 key-value형태로 저장되어있음

## 자료구조

1. 컨테이너 map 설명 + 시간복잡도에 대해 설명해주세요.

map은 key-value쌍으로 이루어진 자료구조. 내부적으로 레드블랙트리로 구현되어있고 key값을 기준으로 정렬이 된다. 삽입, 삭제, 탐색이 O(log N)

1-1. unordered_map 비교

얘는 내부적으로 해시테이블로 되어있다. 정렬이 이루어지지 않음. 삽입,삭제,탐색이 O(1), 해시충돌 최악의 경우 O(N) 까지

1-2. 레드블랙트리 vs 균형이진탐색트리

레드블랙트리도 균형이진탐색트리이긴 하나 대표적인 AVL트리와는 차이가 있다. 그냥 성능과 안정성의 트레이드 오프로 레드블랙트리를 컨테이너에서 많이 사용한다.

1-3. 해시 충돌이 잦으면 unorered_map 성능 변화

해시 충돌이 잦으면 성능이 매우 떨어짐 크기를 더 늘려야 함 Rehasing과정

2. 해시 테이블의 동작원리

해시 테이블은 우선 해시 함수가 필요함. 어떤 값이 들어오면 이를 버킷을 통해서 인덱스로 변환을 해야 한다. 그 상태로 저장해놓음. 값이 많아지거나 수용량이 적으면 충돌이 날 수있는데 이때 성능이 매우 저하됨. 여러 알고리즘 같은것들로 충돌을 방지하거나 그렇다.

- Chaining : 같은 해시 값이 나오면 연결리스트나 트리로 줄세우는 방식
- Open Addressing : 충돌이 나면 옆의 빈 칸을 찾아가는 방식

3. 멀티 스레드 환경에서 안정적으로 사용할 수 있는 자료구조

Thread-safe, 여러 스레드가 한 자료구조에 동시에 접근하게 되면 오류가 발생할 수 있는데 Mutex나 read-write lock을 사용하여 데이터 접근을 제어하거나, 아예 락을 걸지 않고도 lock-free를 사용해야 함(std::atomic)
