## 언리얼 엔진의 DirtyFlag

언리얼 엔진은 성능 최적화를 위해 모든 변경사항을 즉시 디스크에 저장하지 않음. 대신 오브젝트가 더티(Dirty)상태 인지 추적하고, 더티 상태일 때만 저장작업을 수행함. MarkDirty()는 이 더티 플래그를 수동으로 세우는 함수

1. UObject::MarkPackageDirty()
   에디터에서 에셋을 수정했음을 엔진에 알릴때 사용. 이걸 호출하지 않으면 에디터가 변경사항을 인식하지 못해 저장 여부를 물어보지 않음

2. FProperty/Replication 관련 MarkDirty()
   네트워크 복제 환경에서 특정 프로퍼티가 변경되었음을 복제 시스템에 알릴 때 사용.

# TIL: Lyra의 Fragment 패턴 — 클래스 안 건드리고 데이터 확장하기

Lyra 소스 분석하다가 인벤토리 아이템 정의 구조가 생각보다 훨씬 유연하게 설계되어 있어서 놀랐다.

보통 아이템 정의 클래스 만들면 이런 식으로 짠다:

```cpp
class UItemDefinition : public UObject
{
    FText DisplayName;
    int32 MaxStack;
    float Weight;
    TSubclassOf<UEquipment> EquipmentClass;  // 장착 아이템이면
    int32 Damage;                             // 무기면
    float HealAmount;                         // 소비 아이템이면
    // ... 계속 늘어남
};
```

아이템 종류 늘어날수록 클래스가 비대해지거나, 상속 계층이 깊어지거나, 쓰지도 않는 필드가 가득 차게 된다.

## Lyra의 해결 방법: Fragment

```cpp
// 추상 베이스
UCLASS(DefaultToInstanced, EditInlineNew, Abstract)
class ULyraInventoryItemFragment : public UObject
{
    virtual void OnInstanceCreated(ULyraInventoryItemInstance* Instance) const {}
};

// 아이템 정의는 그냥 Fragment 배열만 들고 있음
UCLASS(Const, Abstract)
class ULyraInventoryItemDefinition : public UObject
{
    UPROPERTY(EditDefaultsOnly, Instanced)
    TArray<TObjectPtr<ULyraInventoryItemFragment>> Fragments;
};
```

Fragment 구현체는 각자 필요한 데이터만:

```cpp
// 장착 가능 여부 → 이 Fragment 붙이면 됨
class UInventoryFragment_EquippableItem : public ULyraInventoryItemFragment
{
    TSubclassOf<ULyraEquipmentDefinition> EquipmentDefinition;
};

// 초기 스탯 세팅 → 이 Fragment 붙이면 됨
class UInventoryFragment_SetStats : public ULyraInventoryItemFragment
{
    TMap<FGameplayTag, int32> InitialItemStats;

    virtual void OnInstanceCreated(ULyraInventoryItemInstance* Instance) const override;
    // → 인스턴스 생성 시점에 StatTag 자동 세팅
};
```

에디터에서는 `Instanced + EditInlineNew` 덕분에 배열에 Fragment를 드롭다운으로 추가하고 바로 값 편집 가능.
새 Fragment 클래스 만들어도 `ULyraInventoryItemDefinition` 코드는 한 줄도 안 건드린다.

---

## 핵심 UCLASS 지정자 조합

| 지정자               | 역할                                                   |
| -------------------- | ------------------------------------------------------ |
| `DefaultToInstanced` | 이 클래스의 UObject 포인터는 기본적으로 Instanced 취급 |
| `EditInlineNew`      | 에디터에서 인라인으로 새 오브젝트 생성 허용            |
| `Abstract`           | 직접 인스턴스 생성 불가 (구현체만 사용)                |

UPROPERTY 쪽 `Instanced` 키워드가 없으면 에디터 인라인 편집이 안 된다는 것도 세트로 알아두면 좋다.
