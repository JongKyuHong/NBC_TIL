## RVO

RVO는 움직이는 객체들이 서로의 속도와 방향을 고려하여 충돌을 피하는 방법

```c++
GetCharacterMovement()->bUseRVOAvoidance = true;
GetCharacterMovement()->AvoidanceConsiderationRadius = 200.0f;
GetCharacterMovement()->AvoidanceWeight = 0.5f;
```

- bUseRVOAvoidance : RVO충돌 알고리즘 활성화 On/Off
- AvoidanceConsiderationRadius : AI가 다른 오브젝트를 감지하고 회피를 시작하는 거리를 결정
- AvoidanceWeight : 해당 클래스를 상속받은 캐릭터의 회피 우선순위 결정

강의에서는 병목현상을 만들고 RVO를 통해서 AI가 서로 비켜가는것을 보였지만 만약 RVO가 없다면 AI는 지나가지를 못함(경로상의 장애물을 피해가지 못함)

## AI Controller & BT & BB

1. AI Controller
   조종사, Pawn 소유 제어, AI Perception으로 소리감지, 적을봄 과같은 감각 정보 수집, BT를 구동시키고 BB를 할당

2. BlackBoard
   메모장, 데이터 저장 (AI가 알고있는 정보를 Key-Value형태로 저장), 공유 메모리 역할(AI Controller가 감각으로 얻은 정보를 여기에 적으면 BT가 이 정보를 보고 판단함)

3. BT
   행동 지침서, 타겟이 있는가? -> 있으면 공격 없으면 순찰과 같은 복잡한 흐름 제어, 계층구조, 상태 체크

### 서로 어떻게 협업?

1. 정보 수집 및 기록 (AI Controller -> BB)
2. 상황 판단 (BB -> BT) : BT는 매 틱 혹은 이벤트 발생 시 로직을 수행, BT가 BB를 확인함
3. 행동 결정(BT)
4. 명령 수행 (BT -> AI Controller -> Pawn)

## 위상 정렬

- 방향성이 있고, 사이클이 없는 그래프에서 모든 정점을 순서에 맞게 나열하는 알고리즘
- 쉽게말해 일의 우선순위, 선후관계가 정해져있을때 이를 어기지않고 순서를 짜는 방법
- 그래프에 사이클이 있다면 시작점을 찾을 수 없으므로 정렬을 수행할 수 없다.
- 진입차수가 중요한 개념이다. 진입차수가 0이면 더 이상 수행할 사전작업이 없다는 뜻으로 이를 큐에넣고 돌림

```c++
#include <iostream>
#include <vector>
#include <queue>

using namespace std;

int n, m;
int indegree[100001];
vector<int> adj[100001];

void topologicalSort(){
    vector<int> result;
    queue<int> q;

    for (int i = 1; i <= n; i++){
        if (indegree[i] == 0) q.push(i);
    }
    while (!q.empty()){
        int cur = q.front();
        q.pop();
        result.push_back(cur);

        for (int next : adj[cur]){
            indegree[next]--;
            if (indegree[next] == 0) q.push(next);
        }
    }
    for (int x : result) cout << x << " ";
}
```

## 세그먼트 트리

- 특정 구간의 합, 최솟값, 최대값등을 아주 빠르게 구하고, 데이터의 업데이트도 효율적으로 처리하기 위해 사용하는 자료구조
- 세그먼트 트리는 기본적으로 완전 이진트리 형태를 띔
- 트리를 생성 (보통 배열 크기의 4배의 높이로) -> 재귀적으로 왼쪽 자식과 오른쪽 자식의 값을 더해 부모 노드를 채워나감(조건에따라 다를수도) -> 구간 쿼리 (우리가 구하고자 하는 구간과 현재 노드가 담당하는 구간을 비교)
- 구간 쿼리 : 구간이 겹치지 않으면 결과에 영향을 주지않는 값 합이라면 0, 최솟값이라면 무한대 반환, 구간이 포함되면 현재 노드값을 반환, 구간이 일부 겹쳐있으면 자식으로 내려가서 다시 탐색하고 합침
- 특정 인덱스 값을 바꿀때는 해당 리프노드부터 시작해 루트까지 올라가며 경로에있는 모든 부모노드들을 갱신함

```c++
#include <vector>
#include <cmath>

using namespace std;

long long tree[400000]; // N이 10만일 때 약 4배 크기
long long arr[100000];

long long init(int node, int start, int end){
    if (start == end) return tree[node] = arr[start];
    int mid = (start+end) / 2;
    return tree[node] = init(node*2, start, mid) + init(node*2+1, mid+1, end);
}

long long sum(int node, int start, int end, int L, int R){
    if (R < start || end < L) return 0; // 범위를 벗어난 경우
    if (L <= start && end <= R) return tree[node]; // 범위에 포함된 경우
    int mid = (start + end) / 2;
    return sum(node*2, start, mid, L, R) + sum(node * 2 + 1, mid + 1, end, L, R);
}

void update(int node, int start, int end, int idx, long long diff){
    if (idx < start || idx > end) return;
    tree[node] += diff;
    if (start != end){
        int mid = (start + end) / 2;
        update(node*2, start, mid, idx, diff);
        update(node*2+1, mid + 1, end, idx, diff);
    }
}
```

## MST

- 그래프 내의 모든 정점을 가장 적은 비용으로 연결하기 위해 사용하는 알고리즘
- 모든 노드가 연결되어야하지만 사이클이 있어서는 안됨!

### 크루스칼 알고리즘

- 간선 중심의 그리디한 알고리즘
- 가장 가벼운 간선부터 차례로 연결
- 간선을 가중치 기준으로 오름차순 -> 정렬된 간선을 하나씩 확인해서 그래프에 추가 -> 사이클이 발생하는지 확인 (Union-Find) -> 모든 간선 확인이 끝나거나 간선의 개수가 V-1개가 되면 종료

```c++
struct Edge{
    int u, v, weight;
    bool operator<(const Edge& other) const{
        return weight < other.weight;
    }
}

int findParent(int x){
    if (parent[x] == x) return x;
    return parent[x] = findParent(parent[x]);
}

// 메인로직 예시
sort(edges.begin(), edges.end());
for (auto& edge: edges){
    if (findParent(edge.u) != findParent(edge.v)){
        unionNode(edge.u, edge.v);
        totalCost += edge.weight;
    }
}
```

### 프림 알고리즘

- 내가 가진 네트워크에서 가장 가까운 곳을 하나씩 포섭한다.
- 우선순위큐를 사용하여 구현, 현재 방문한 정점들에서 갈 수 있는 간선들을 큐에 넣고 관리
- 임의의 정점에서 시작 -> 큐에서 가중치가 가장 작은 간선 꺼냄(가장 가까운 노드 선택) -> 간선이 연결하는 노드가 이미 MST에 포함되어 있다면 버림 -> 아직 방문하지 않은 노드라면 MST에 포함시키고, 그 노드에서 갈 수 있는 다른 간선들을 모두 큐에 넣음

```c++
#include <iostream>
#include <vector>
#include <queue>

using namespace std;

typedef pair<int, int> pii;
vector<pii> adj[10001];
bool visited[10001];

int prim(int startNode){
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    int totalWeight = 0;

    pq.push({0, startNode});

    while (!pq.empty()){
        int cost = pq.top().first;
        int ur = pq.top().second;
        pq.pop();

        if (visited[cur]) continue;

        visited[cur] = true;
        totalWeight += cost;

        for (auto& edge : adj[cur]){
            int nextCost = edge.first;
            int nextNode = edge.second;
            if (!visited[nextNode]){
                pq.push({nextCost, nextNode});
            }
        }
    }
    return totalWeight;
}
```

### 레드 블랙 트리

- 임의 노드에서 그 자손인 리프 노드까지 갈때 모든경로에서 블랙노드의 개수가 같아야함
- 빨간 노드는 이어지면 안됨 (이덕에 항상 재배치됨)
- 루트는 항상 검은색

1. 데이터 추가 : 빨간노드로 들어옴 규칙에 따라 회전이 발생 (루트가 빨간색이거나, 빨간리본이 연달아 나오거나)
2. 데이터 삭제 : 검은 노드가 사라지면 같은 깊이의 검은노드 수 규칙이 깨질수도있음

- 자동 정렬 : O(log N)이하로 제한됨

## 포인터와 참조의 차이점

- 포인터는 어떤 변수의 메모리 주소값을 들고 있는 별도의 변수
- 참조는 이미 존재하는 변수에 붙이는 또다른 별명
- 참조는 반드시 초기화 해야함, 재할당이 불가, 별도의 공간을 차지하지 않음, 변수처럼 직접 접근 가능
- 포인터는 나중에 초기화해도됨, 언제든 다른 주소 가리킬 수 있음, 별도의 메모리 공간 차지(주소 저장용), -> 또는 \*를 사용하여 역참조 해서 접근
- 게임개발에서는 포인터는 nullptr일수도있는데 참조는 원본값이므로 보장이됨
- 큰 구조체 넘길때 값을 통째로 복사하지않고 이름만 빌려주는식이라 성능이 좋다.

- int& ref = a : 참조자, 레퍼런스, a의 별명을 만듬
- &a : 주소 연산자, address of operator, a의 메모리 주소값

## C++ lvalue와 rvalue

- lvalue는 이름이 있는 변수, 메모리 상에서 주소를 가지고 있어 다음줄에서도 쉽게 접근 가능, &(주소 접근자)를 사용해서 주소값을 알아낼 수 있다면 lvalue
- rvalue 이름이 없는 임시 값, 표현식이 끝나면 사라지는 녀석, 메모리 주소를 고정적으로 점유하지 않음, &연산자를 쓸 수 없다.
- 무슨 ravlue참조(&&)라는 개념이 또 있다고 하더라, 이동 생성자와 이동 대입 연산자의 핵심으로 곧 버려질 rvalue의 자원을 새로 메모리 할당하지 않고 이동시켜 성능적인 향상이 있다고 한다.

```c++
#include <iostream>
#include <utility> // std::move를 위해 필요

class MyBuffer {
private:
    int* data;
    size_t size;

public:
    // 생성자
    MyBuffer(size_t s) : size(s) {
        data = new int[size];
        std::cout << "자원 할당 (생성자)\n";
    }

    // 소멸자
    ~MyBuffer() {
        delete[] data;
        std::cout << "자원 해제 (소멸자)\n";
    }

    // [1] 복사 생성자 (깊은 복사: 비쌈)
    MyBuffer(const MyBuffer& other) : size(other.size) {
        data = new int[size];
        for (size_t i = 0; i < size; ++i) data[i] = other.data[i];
        std::cout << "복사 생성자 호출 (깊은 복사)\n";
    }

    // [2] 이동 생성자 (자원 탈취: 매우 빠름)
    // &&는 rvalue 참조입니다. '곧 사라질 녀석'임을 의미해요.
    MyBuffer(MyBuffer&& other) noexcept : data(nullptr), size(0) {
        // 1. 상대방의 주소값을 내 것으로 가져옴
        data = other.data;
        size = other.size;

        // 2. 상대방의 포인터를 NULL로 만들어 '연결 고리'를 끊음 (중요!)
        other.data = nullptr;
        other.size = 0;
        std::cout << "이동 생성자 호출 (자원 뺏기)\n";
    }

    // [3] 이동 대입 연산자
    MyBuffer& operator=(MyBuffer&& other) noexcept {
        if (this != &other) {
            delete[] data; // 기존 내 자원 정리

            data = other.data; // 상대방 거 뺏기
            size = other.size;

            other.data = nullptr; // 상대방 거 초기화
            other.size = 0;
            std::cout << "이동 대입 연산자 호출\n";
        }
        return *this;
    }
};
```
