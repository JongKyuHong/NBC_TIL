## 언리얼에서 Interface를 쓰는 상황

- 상호작용의 일반화를 위해
- 아이템, 문, NPC가 있을때 이는 모두 캐릭터와 상호작용이 가능하다. 하지만 부모 클래스가 전혀 다름
- 이때 인터페이스가 없다면 플레이어 코드에서 일일이 Cast를 해줘야한다.
- 클래스가 100개라면 캐스팅도 100번해야함

```c++
// U는 언리얼 엔진에게 "이것은 인터페이스다"라고 알려주는 등록증과 같습니다.
UINTERFACE(MinimalAPI, Blueprintable)
class UInteractiveObject : public UInterface
{
    GENERATED_BODY()
};

// I는 실제 계약서로, "이 인터페이스를 구현하는 모든 클래스는 이런 기능을 반드시 가져야 한다"라는 명세서입니다.
class IInteractiveObject
{
    GENERATED_BODY()

public:
    // 상호작용 메서드 (인터랙션이 일어나는 상대 오브젝트에서 반드시 구현해야 함)
    UFUNCTION(BlueprintNativeEvent, BlueprintCallable, Category = "Interaction")
    void Interact(AActor* Interactor);
};
```

- 인터페이스를 호출할때는 Execute\_ 접두사가 붙은 정적함수를 사용해야 한다.
- 그냥 Cast<IInteractableInterface>(TargetActor)를 해서 호출할 수도 있지만 이렇게하면 블루프린트에서만 인터페이스를 구현한 액터는 호출하지 못함

# 개념

## 순열

- 정렬상태여야 함
- 주어진 수열의 모든 순열 생성, 사전순으로 특정 순열 찾기 같은 문제에 유용
- permutation관련 문제를 만났을때 next_permutation, prev_permutation을 쓰면 된다.

```c++
int main() {
    vector<int> v = {1, 2, 3};

    // 다음 순열 구하기
    for (int x : v) cout << x << " "; // 1 2 3
    cout << endl;

    next_permutation(v.begin(), v.end());

    for (int x : v) cout << x << " "; // 1 3 2
    cout << endl;

    // 모든 순열 구하기
    do {
        for (int x : v) cout << x << " ";
        cout << endl;
    } while(next_permutation(v.begin(), v.end())); // 다음순열이 존재하지 않는 한 계속 탐색
}
```

- 순열의 `이전`과 `다음`은 사전순으로 정의됨

## K번째 값 찾기

- nth_element를 사용, pivot기반으로 n번째 작은 원소를 배열의 n번째 위치로 보낸뒤 그 왼쪽은 n번째보다 작은 원소들, 오른쪽은 n보다 큰원소들 이런식으로 반정렬을 해줌
- nth_element는 O(N)의 시간복잡도

```c++
int main() {
    vector<int> v = {9, 1, 4, 7, 2, 6};

    nth_element(v.begin(), v.end()+2, v.end());
    cout << "3번째로 작은 원소 : " << v[2] << endl;
}
```

- 중간값을 찾는다면?

```c++
nth_element(v.begin(), v.begin() + v.size()/2, v.end());
```

## 스택과 힙 메모리 영역의 차이

- 스택은 지역변수와 매개변수가 저장, 함수가 종료되면 자동으로 메모리가 해제, 속도가 빠르지만 크기가 제한적
- 힙은 개발자가 new나 malloc같이 직접 메모리를 할당하는 영역, 사용자가 해제하지 않으면 메모리 누수가 발생한다. 스택보다 훨씬 크지만 할당/해제 속도가 상대적으로 느리다.

# 알고리즘

## 공원

```c++
// https://school.programmers.co.kr/learn/courses/30/lessons/340198

#include <string>
#include <vector>
#include <algorithm>

using namespace std;

int solution(vector<int> mats, vector<vector<string>> park) {
    int answer = -1;

    int N = park.size();
    int M = park[0].size();
    vector<vector<int>> dp(N, vector<int>(M, 0));

    for (int i = 0; i < N; ++i){
        for (int j = 0; j < M; ++j){
            if (park[i][j] == "-1") {
                if (i == 0 || j == 0) {
                    dp[i][j] = 1;
                    continue;
                }
                dp[i][j] = min({dp[i-1][j], dp[i][j-1], dp[i-1][j-1]}) + 1;
            }
        }
    }

    int maximum = 0;
    for (int i = 0; i < N; ++i){
        for (int j = 0; j < M; ++j){
            maximum = max(maximum, dp[i][j]);
        }
    }

    for (int mat : mats){
        if (maximum >= mat){
            answer = max(answer, mat);
        }
    }

    return answer;
}
```

- dp로 잘푼것같아서 기억하려고
- 정사각형이려면 (i, j-1), (i-1, j), (i-1, j-1)을 다 확인해야 하고, 그들의 최솟값을 통해서 갱신하면 된다.
