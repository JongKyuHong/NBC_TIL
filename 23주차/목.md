## inline함수는 언제 사용되며 실제로 어떻게 처리되는가?

inline함수는 최적화를 위해 설계된 도구, 함수 호출 지점에 함수의 본문 코드를 직접 삽입하도록 컴파일러에게 요청

1. 컴파일러의 실제 처리 방식
   호출할 함수의 주소로 점프 -> 현재 함수의 상태를 스택에 저장 (레지스터, 복귀 주소) -> 함수의 매개변수를 스택이나 레지스터에 복사 -> 함수 실행 완료 후 다시 원래 위치로 복귀

함수에 inline이 적용되면, 컴파일러는 호출 부하를 없애기 위해 다음과 같이 처리함

코드 치환 -> 컴파일러는 함수를 호출하는 명령문 대신 함수의 본문 전체를 호출 지점에 복사

```c++
class Player {
public:
    // 클래스 내부 구현 -> 암시적 인라인 함수
    int GetHealth() const { return Health; }

private:
    int Health;
};
```

- 클래스 선언부 (헤더)에서 함수본문을 직접 구현하면 암시적으로 inline으로 처리된다고 한다.

## 함수 포인터 (Function Pointer)

함수 포인터는 함수의 주소를 저장하는 변수, 특정 함수를 직접 호출하는 대신, 실행 중에 호출할 함수를 결정할 수 있게 해줌

```c++
// 간단한 산술 함수들
int Add(int a, int b) { return a + b; }
int Multiply(int a, int b) { return a * b; }

// 함수 포인터를 매개변수로 받는 함수
void ExecuteOperation(int x, int y, int (*operation)(int, int)) {
    std::cout << "Result: " << operation(x, y) << std::endl;
}

int main() {
    // 1. 함수 포인터 선언 및 할당
    int (*funcPtr)(int, int) = Add;
    std::cout << "Add: " << funcPtr(10, 20) << std::endl;

    // 2. 다른 함수로 교체
    funcPtr = Multiply;
    std::cout << "Multiply: " << funcPtr(10, 20) << std::endl;

    // 3. 콜백(Callback) 형태로 사용
    ExecuteOperation(5, 4, Add);

    return 0;
}
```

### 사용하는 이유

- 콜백 구현 : 특정 이벤트가 발생했을때 호출될 함수를 등록할 수 있음
- 전략 패턴 : 동일한 인터페이스로 상황에 따라 다른 알고리즘을 실행할 수 있음
- 동적 바인딩 : 컴파일 시점이 아닌 런타임에 실행할 로직을 교체하여 유연성을 높임
